# AI Talk Topic B 认领准备方案（设计稿）

**日期**：2026-04-18（周六）
**作者**：宋姝（自用规划）
**目标场景**：曹丽娜在「与AI共进」群发起的下周五（4/24）AI Talk，认领 Topic B「AI 技术演进探讨」

---

## 1. 背景

- **群**：与AI共进（18 人），曹丽娜（直属领导）发起
- **AI Talk 信息**：
  - 时间：4/24 周五 19:30–21:30（共 120 min，两个 topic 共享）
  - 地点：F10-绿松石会议室
  - 形式：每个 topic 一人主讲 + 大家探讨
- **可认领的两个 topic**：
  - A. 模型架构相关探讨（Transformer vs CNN / NPU 跑 Transformer 瓶颈 / 提高 NPU 利用率）
  - B. AI 技术演进探讨（RNN/CNN → ResNet → Transformer → LLM → Diffusion → VLM → World Models → GEN-1）
- **截图观察**：截至 4/18 群里只有点赞，**无人公开认领**

## 2. 核心决策

### 2.1 目标与优先级

**核心目的**：帮领导分担、支持领导拉起的 topic。

**成功标准优先级**：
1. **a 兜底**（最重要）：稳稳撑住、不冷场
2. **b 立人设**：让领导和团队感知到我有 AI 方向的沉淀
3. **c 自我升级**：借机系统梳理 AI 演进脉络，对自己 SLAM/3DGS 主业的未来方向有帮助

### 2.2 选 topic：B

**起点自评**：A、B 都是 L1（完全不熟）

**理由**（对齐 a > b > c）：
- **A 的翻车风险高**：NPU/部署是硬核工程，群里大概率有人真做过，被追问就接不住
- **B 容错度高**：脉络梳理是叙事型，深度可控，听众预期就是"串一条线"
- **B 立人设抓手好**：可以输出"我的判断框架"，框架性输出是 leader 视角
- **B 对主业更对口**：后半段的 Diffusion / VLM / World Models 与 3DGS、世界模型方向相关

### 2.3 分享形式：偏分享型

**讲 40 min + 讨论 15 min**

- 主动权在自己，不依赖现场气氛
- 立人设信号比"引导讨论"更强
- **代价**：L1 起步 + 2 天准备 + 40 min 硬内容，强度高，准备不足要敢于止损

### 2.4 决策节奏：周末试准备 → 周日晚定 Go/No-go

- 周末 2 天做"试准备"，周日 21:00 前**自评到框架级 50%** 才正式认领
- 周末**不在群里发声**，避免硬接后撤不回
- **No-go 备选**：(γ) 完全退出本次，承诺下次 AI Talk 提前 2 周准备

### 2.5 框架级 50% 的定义

一份大纲：
- 一级标题（4 个） + 每个一级下 3–4 个二级要点
- 每个要点能讲 2–3 句
- 时长分配明确

## 3. 周末 2 天行动计划

### 3.1 全景图

```
周六（6h 内容 + 中间休息）
  09:00-12:00  横向扫描：8 节点速查表        → 8 张速查卡（25%）
  12:00-14:00  午饭 + 休息
  14:00-15:30  找叙事主线（3 个 Q × 30min）  → 2-3 条主线（35%）
  15:30-17:00  搭大纲 v1                      → 大纲 v1（50%）
  17:00 后    断开，不再做内容（强制）

周日（5h 内容 + 中间休息）
  09:00-12:00  填二级要点 → 大纲 v2          → 大纲 v2（75% 框架）
  12:00 中期检查点（见 3.4）
  12:00-14:00  午饭 + 休息
  14:00-15:30  空讲 + 录音
  15:30-16:00  卡壳清单分类                  → 大纲 v3 + 清单（90%）
  16:00-20:00  休息 / 散步 / 让大脑沉淀
  20:00-21:00  Go/No-go 判定（见 3.5）       → 决策（100%）
  21:00 后    根据决策准备周一动作
```

### 3.2 周六上午：8 节点速查表

每个节点抓 4 件事：

| 字段 | 要回答 | 字数 |
|---|---|---|
| 是什么 | 一句话定义 | 1 句 |
| 解决了什么 | 之前卡在哪？解决了哪个痛点？ | 2 句 |
| 代表作 | 1–2 个论文/模型/事件 + 年份 | 1 行 |
| **我的一句判断** | 用自己的话总结这一步在 AI 史里的意义 | 1 句 |

**"我的一句判断"是立人设的核心弹药**——L1 起步没办法在细节上压人，但每个节点都有自己的判断 = 听众觉得"宋姝是有视角的，不是搬运工"。

**8 节点时长分配**（不均匀，按重要性）：

| # | 节点 | 时长 | 重点 / 避坑 |
|---|---|---|---|
| 1 | RNN / CNN | 20min | 别陷 LSTM 门控细节 |
| 2 | ResNet | 15min | 重点是"残差连接让网络可以做深" |
| 3 | **Transformer** | **30min** | 重点投资。不要讲 Q/K/V 公式 |
| 4 | LLM | 25min | 重点是 scaling law + 涌现，不讲 RLHF 细节 |
| 5 | Diffusion | 25min | 不讲 DDPM 推导，讲清"加噪→去噪"思路 |
| 6 | VLM | 20min | 重点是"视觉语言对齐到同一表征空间" |
| 7 | World Models | 20min | 最容易讲虚，必须落到具体例子（Genie / Sora 争论） |
| 8 | GEN-1 | 15min | 节点窄，讲清"它是 video diffusion 代表作" |

合计 170min，留 10min 机动。

**严格控时**：到点就停，不完美也走。**不要打开第二篇论文**。

### 3.3 周六下午：找叙事主线 + 大纲 v1

**前 90 min — 找叙事主线**（每个 Q 30min）：

- Q1：8 步里"表征方式"经历了什么变化？
- Q2：8 步里"驱动力"是什么在变？
- Q3：8 步里"AI 能做什么任务"在怎么扩大？

挑出**写得最顺的 2 条**作为分享主轴，第 3 条作为讨论 hook。

**后 90 min — 搭大纲 v1**（按"4 阶段范式跃迁"重组 8 节点）：

```
开场（3 min）
  - 这场聊什么 / 不聊什么
  - 我的视角：从 [主线 1] 和 [主线 2] 两条线串

主体（32 min）
  - 第一阶段：判别式 AI 的成熟（RNN/CNN + ResNet）         6 min
  - 第二阶段：Transformer 范式的崛起（Transformer + LLM）   10 min  ← 重点段
  - 第三阶段：生成式 AI 的爆发（Diffusion + VLM）           8 min
  - 第四阶段：走向世界模型与具身（World Models + GEN-1）    8 min

收尾（5 min）
  - 我的整体判断
  - 抛 3 个值得讨论的问题，引入 15 min 讨论
```

### 3.4 周日上午：填二级要点 + 中期检查

**逐个一级标题填充**，每个不超过 45min（4 × 45 = 180min）。

**每个二级要点的写法**：关键词 + 我要讲的 2–3 句。写不出来的标 ⚠️ 跳过。

**关键原则**：
- 每个一级段最后必须有一个"我的判断"二级要点（4 个段 = 4 次输出判断的机会）
- 二级要点不超过 4 个，超过说明切太细

**12:00 中期检查（早预警）**：

| ⚠️ 待补数 | 含义 | 行动 |
|---|---|---|
| ≤ 5 处 | 框架立得很稳 | 按计划下午空讲 |
| 6–8 处 | 框架基本可用，有薄弱段 | 按计划下午空讲，重点关注薄弱段 |
| **≥ 9 处** | **框架没立稳** | **不硬撑做空讲，直接进入决策**：承认 No-go 或降级到 b 引导型 |

### 3.5 周日下午：空讲 + 卡壳清单

**空讲（必须录音）**：
- 站起来 / 关电脑屏幕只看大纲打印稿 / 录音 / 计时器
- 讲错卡住**继续往下**，不要回头修正
- 每个一级段记录"实际 X min vs 计划 Y min"

**卡壳清单分类**：

| 类型 | 含义 | 处理 |
|---|---|---|
| **A 内容空洞** | 根本没想清楚要讲什么 | 必须补，最高优 |
| **B 表达不顺** | 内容对，话术绕 | 标记后续磨，本次不补 |
| **C 超时** | 切太细 / 内容太多 | 砍要点 |
| **D 欠时** | 太薄 / 撑不起时长 | 加要点或调时长 |

**产物**：大纲 v3 + 卡壳清单 + 实际总时长数字

### 3.6 周日 20:00–21:00：Go/No-go 判定

**4 项硬指标**：

| # | 指标 | 不达标含义 |
|---|---|---|
| 1 | 大纲完整：4 个一级 + 3–4 个二级，⚠️ ≤ 3 处 | 框架没立稳 |
| 2 | 空讲跑完，无整段塌方 | 这段实际是空的 |
| 3 | 总时长 30–50 min（计划 40，±25%） | 时长失控 |
| 4 | A 类卡壳 ≤ 3 处 | 没准备好 40 min 内容 |

**判定**：
- 4 ✅ → **Go**：周一群里认领，下周一到周四磨 ⚠️ 和 B 类卡壳
- 3 ✅ + 1 ❌ → 谨慎 Go：失守 1/2/4 倾向 No-go；失守 3 → Go（时长是工艺问题）
- ≤ 2 ✅ → **No-go**

**No-go 备选：(γ) 完全退出本次**，承诺下次 AI Talk 提前 2 周准备。

> 选 γ 的理由：α/β 都是"半接半推"，对自己和领导都不够干净；γ 最坦诚，且"提前 2 周准备"是更可持续的承诺，比这次硬上半成品更能立长期人设。

## 4. 关键设计原则

1. **广度优先，深度后置**——周末只解决"能不能讲完 40 min"，深度是下周的事
2. **大纲驱动，不写逐字稿**——L1 起步写逐字稿是陷阱
3. **空讲是硬指标**——不空讲不知道是真懂还是错觉懂
4. **周六晚强制断开**——疲劳会让周日效率掉一半
5. **早预警优先于晚决策**——周日上午 12:00 中期检查比周日晚 21:00 决策更划算
6. **敢于 No-go**——准备不到位硬上的代价 > 这次不接的代价

## 5. 周一动作（待 Go/No-go 后填）

**Go 路径**：在群里认领 B，附大纲简介给曹丽娜看方向。

**No-go (γ) 路径**：在群里说明本次不接，承诺下次提前准备。措辞先不预演，等真到那一步根据准备过程的实际感受来写，会更真诚。
