# AI Talk Topic B 投屏用图文讲义（Markdown）设计

**日期**：2026-04-26  
**关联材料**：`projects/personal-ai-talk-2026-04-24/05-full-script.md`（逐字稿）、`09-fact-check-log.md`（事实锚点）、`README.md`（文件清单与已锁定决策）  
**产出物**：新建 `projects/personal-ai-talk-2026-04-24/08-projection-handout.md`

---

## 1. 背景与问题

逐字稿 `05-full-script.md` 面向「念 + 讲」，含串场、节奏标记、口语化说明；组会投屏时观众需要的是**高密度信息板**：图为主、文字为辅，与口播解耦。

本设计定义一份**单文件 Markdown 图文讲义**，覆盖与逐字稿一致的**完整 40 分钟结构**（4 老阶段 + 3 延伸 + 开场 + 收尾讨论），每节压缩为「图示建议 + 核心 bullet + 一句判断」。

---

## 2. 目标与非目标

### 2.1 目标

- **投屏可读**：单屏信息量少、层级清晰；观众不依赖口播也能抓住本节要点。
- **与逐字稿一一对应**：章节顺序与 `05-full-script.md` 主线一致，便于演讲时「左讲义 / 右逐字稿」或上下分屏。
- **事实以核查日志为准**：所有时间、数字、产品名与 `09-fact-check-log.md` 对齐；讲义中不引入逐字稿未锚定的新事实。
- **视觉占位明确**：每节有 `[图示建议：…]`，说明应配图类型与内容要点；已有素材（如 `assets/main-timeline.svg`）在总览节直接 `![…](…)` 嵌入。

### 2.2 非目标

- **不替代** `05-full-script.md`（不删除、不重写逐字稿）。
- **不承诺**在本迭代内补齐全部真实配图文件；以 Markdown 占位 + 路径引用为主，后续你可按需补图。
- **不生成** Keynote/PPT/Marp 工程；若未来需要，可另开任务从本 Markdown 导出。

| 项    | 决定                                                              |

## 4. 文件结构与模板

### 4.1 文件头部（建议固定块）

```markdown
# AI 技术演进探讨 · 投屏讲义

> 用途：组会投屏；事实以 `09-fact-check-log.md` 为准。

## 讲次结构速览

（总览图或时间轴嵌入此处）
```

### 4.2 每一节的固定骨架

```markdown
## N. 节标题（与逐字稿大段同名）

> 一句话本节要传递的信号（可选，≤ 一行）

**[图示建议]**：一句话说明本屏应出现的图（类型 + 关键元素）。

![占位说明](assets/xxx.png)  <!-- 有图则嵌入；无图则整段删除或保留图示建议块 -->

- 核心信息 1（短句，尽量可扫读）
- 核心信息 2
- 核心信息 3

**我的判断**：一句收束（与逐字稿判断一致，书面压缩）。
```

**约定**：

- 二级标题 `##` 对应逐字稿的一级叙事单元（开场 / 各阶段 / 各延伸 / 收尾）。
- 三级标题 `###` 仅用于单节内再拆「例子 / 国内副线 / SLAM 视角」等子块，避免过深。
- 不使用 `[节奏]`、`各位晚上好` 等口播专用标记；串场句一律不写入讲义。

### 4.3 章节清单（须全部出现，顺序固定）

1. 开场：基调 + 总览（8 老节点 + 3 新方向时间轴）
2. 第一阶段：判别式 AI（RNN/CNN + ResNet）
3. 第二阶段：Transformer + LLM（含 scaling、ChatGPT、Mamba 补充、2026 frontier 与国内副线）
4. 第三阶段：Diffusion + VLM（CLIP 闭环）
5. 第四阶段：World Models（起源）+ GEN-1
6. 延伸 1：推理大模型（含国内三条线）
7. 延伸 2：具身 VLA（时间线 + 国内 + SLAM 三层栈）
8. 延伸 3：World Models 突破（Genie 3、Cosmos、重建侧 3DGS/DUSt3R/VGGT 对照）
9. 收尾：整体判断 + 国内格局一句 + **Q1–Q3** 讨论问题（编号与逐字稿一致）

## 5. 从逐字稿到讲义的抽取规则

1. **先定图示**：每读完逐字稿一节，把 `[投影：…]` 转写为 `[图示建议]`；若逐字稿无投影标记，补一条最该出现的图。
2. **再抽 bullet**：只保留可验证事实、关键对比、时间线节点；形容词能删则删。
3. **最后写判断**：每节最多一句「我的判断」；与逐字稿立场一致，长度控制在单行或两行内。
4. **数字与时间**：凡写入讲义的具体数字，必须在 `09-fact-check-log.md` 中有对应条目；若无则改为定性表述或删除该数字。

## 8. 风险与缓解


| 风险            | 缓解                                     |
| ------------- | -------------------------------------- |
| 单文件过长导致投屏滚动吃力 | 每节 bullet 上限默认 5 条；超长节用 `###` 拆成 2–3 屏 |
| 图未就绪显得空       | 保留 `[图示建议]`；总览至少一张嵌入 SVG               |
| 与逐字稿两处表述漂移    | 抽取时以 09 为锚；判断句直接从逐字稿「我的判断」压缩           |
