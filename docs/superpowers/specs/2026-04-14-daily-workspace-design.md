## 一、背景与目标

定位组 TL 需要一个**个人日粒度工作空间**，满足两个独立需求：

1. **快速收件**：当天接收的文件（xlsx、pdf、图片等）和随手记录，零摩擦投入，不加工
2. **结构化加工**：有时间时整理为当日日志，覆盖待办、团队进展、排期更新、知识积累

输出仅供个人使用，不同步进现有的 `overview/tasks.md`、`teams/*/people.md` 等团队文档。

---

## 二、目录结构

```
LocalizationTeam/
├── workspace/
│   ├── 2026-04-14/
│   │   ├── inbox/          ← 原始材料，直接拖入，不加工
│   │   └── daily.md        ← 当日加工输出
│   ├── 2026-04-15/
│   │   ├── inbox/
│   │   └── daily.md
│   └── ...
```

**命名规范：**
- 日期目录：`YYYY-MM-DD`
- inbox 文件：保留原始文件名，不重命名
- 输出文件：固定命名 `daily.md`

**与现有工程的关系：**
- `workspace/` 与 `teams/`、`overview/`、`inbox/` 同级
- 语义隔离，不修改现有任何目录，纯增量添加

---

## 三、`daily.md` 模板

```markdown
# YYYY-MM-DD

## 待办
- [ ] ...

## 团队进展
| 成员 | 组 | 进展/状态 | 待跟进 |
|------|---|---------|--------|
|      |   |         |        |

## 排期更新
- [项目/模块] 节点描述 · 日期 · 备注

## 知识 & 信息积累
-

## 原始材料
- [文件名](inbox/文件名) — 来源 · 一句话说明
```

**各 section 说明：**

| Section | 用途 | 填写时机 |
|---------|------|---------|
| 待办 | 当日及近期工作项，checkbox 形式 | 随时 |
| 团队进展 | 1:1、沟通、周报中的成员状态速记 | 有沟通后 |
| 排期更新 | 收到或确认的节点变化、里程碑 | 有新信息时 |
| 知识 & 信息积累 | 技术/管理类值得留存的信息 | 随时 |
| 原始材料 | inbox 文件索引，一句话说明来源 | 投入文件后 |

**填写原则：**
- 所有 section 可以为空，不强制填满
- `团队进展` 是速记，几个字即可，不是周报替代
- `原始材料` 只写索引，内容在 `inbox/` 里

---

## 四、快速创建当日目录

```bash
mkdir -p workspace/$(date +%F)/inbox && touch workspace/$(date +%F)/daily.md
```

在项目根目录执行，一步到位。

---

## 五、设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 位置 | 顶层 `workspace/` | 与 `teams/`、`overview/` 语义隔离，不污染团队文档 |
| 每日粒度 | 一个子目录（含 inbox + daily.md） | inbox 和输出强绑定，同一天的东西在一处 |
| 输出形式 | 单文件 5 section | 翻阅方便，写起来快，不需要跨文件跳转 |
| inbox 命名 | 保留原始文件名 | 投入零摩擦，是第一优先级 |
| 与团队文档关系 | 独立，不同步 | 个人管理日志，和团队结构化文档职责不同 |
