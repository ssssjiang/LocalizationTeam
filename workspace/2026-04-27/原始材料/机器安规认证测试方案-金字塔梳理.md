## 一、项目信息（顶层结论）

**项目主题**：机器安规认证测试方案对齐 — 「定位丢失下机器停轮/停刀」需求

**核心结论**：
- 安规要求是**全部定位失效**时一秒内停轮（≤1m）、5 秒内停刀；**不是**雷达断流时就要停。
- 当前断流 2.0 方案下，激光断流后融合定位仍可由 VIO/IMU/Odom 维持，不应触发安规停机。
- 需要解决的是**测试方案**：如何把所有定位输入都断掉来验证 1m/5s 边界，而不是改产品逻辑。

## 二、目标

| 层级 | 目标 |
|---|---|
| 认证目标 | 通过安规认证：**所有定位信号失效**情况下，机器 ≤1m 停轮、≤5s 停刀 |
| 工程目标 | 由定位组输出"完整定位依赖传感器清单"和**安规测试 SOP** |
| 流程目标 | 建立"安规相关需求/策略变更必须拉认证（钱雪莲）评审"的机制 |
| 兜底目标 | 由导航侧增加「N 帧未收到融合定位 → 立即停机」逻辑作为最后防线 |

---

## 三、关键事实与共识

1. **安规标准的真实含义**：要求是"所有定位都不生效"才需 ≤1m 停机；只要还有一个定位源（VIO/IMU/Odom）能撑住，就不算违反。
2. **定位依赖的传感器**（定位组输出）：
   - 主：激光雷达 + 雷达上的 IMU
   - 辅：Odometry（驱动轮齿轮）+ Gyro（MCU 上）
   - 双目摄像头（VIO）
3. **断流与安规解耦**：安规不应感知"断流"概念，只关心"定位是否还能用"。
4. **导航现状**：未对融合定位做时间/位移监控，定位丢了也会继续走（**风险点**）。
5. **竞品对标**：Versta 等竞品在传感器全遮时也能跑一段距离再停，与产品需求一致。
6. **边界场景特殊性**：靠近边界/被搬动到界外 1m 时，即便辅助定位有偏差也必须停 — 当前定位飘移时**没有主动发起重定位**，仅断流场景会触发。

## 五、工作项（按角色拆分）

### 5.1 定位组（亚萌 / 宋姝 / 思羽）
- [ ] 输出定位依赖的全部传感器清单（雷达、雷达 IMU、双目、Odom、Gyro/MCU IMU）
- [ ] 输出**安规测试 SOP**：如何在 DEV 版本上断开所有定位输入
- [ ] 后续在 monitor 中补充"位移异常 + 时间间隔"双判据
- [ ] 评估非断流场景定位异常主动停机 + 全局重定位策略

### 5.2 导航组（李欢 / 一鸣）
- [ ] 增加「N 帧未收到融合定位 → 立即停机」兜底逻辑（当前完全没有时间判断）
- [ ] 配合中间层接收"机器已行进距离"信号，辅助 1m 内停机判断
- [ ] 评估边界场景下辅助定位失效时的停机策略

### 5.3 中间层
- [ ] 接收导航的位移/状态信号，结合断流判断是否触发停轮/停刀
- [ ] 区分"停轮"与"停刀"两路信号

### 5.4 认证 / 测试（诚武 / 婷雯）
- [ ] 把测试 case 写细：正常 + 异常 + 全信号断开的操作流程
- [ ] 与产品（侯旭 / 杨倩）打合：和现有产品需求冲突点先对齐再下发研发
- [ ] 提交禅道单，跟踪问题至关闭
- [ ] 钱雪莲作为认证侧固定接口人

### 5.5 产品（侯旭 / 杨倩）
- [ ] 在产品经理群明确：涉及安规的需求变更必须拉认证组评审
- [ ] 新功能讨论中 @ 相关认证人，及时反馈，不反馈视为默认通过

### 5.6 电子组
- [ ] 协助确认所有定位传感器到 AP 的物理信号入口（用于测试断信号）

## 六、风险点

| # | 风险 | 影响 | 缓解措施 |
|---|---|---|---|
| 1 | **导航无定位超时兜底**：当前定位 100Hz 发，没收到也不会停 | 真实定位丢失时机器会持续行走，违反安规 | 导航侧加超时停机逻辑（关键兜底） |
| 2 | **测试操作困难**：里程计只要轮子转就有数据，无法物理断开 | 全信号断开测试无法实操 | 抱起机器测试 / 程序级屏蔽 / 拉电子+定位共同设计 SOP |
| 3 | **流程缺位**：早期断流 2.0 评审未拉认证，导致现在被动追问 | 认证晚发现风险，可能漏到市场 | 建立"安规策略变更强制拉认证评审"机制 |
| 4 | **安规需求与产品需求冲突**：安规要求严停，产品要求顺滑 | 改了产品体验，竞品做不到也不停 | 先认证组与产品打合再下研发，避免无用功 |
| 5 | **边界场景未覆盖**：被搬到界外 1m 时辅助定位偏差大但无主动停机 | 越界风险 | 非断流场景定位异常停机+全局重定位策略（待排期） |
| 6 | **特殊版本之争**：诚武想用 DEV 版+断信号；研发不想出特殊版本 | 测试方案不落地 | 不出特殊版本，由定位组输出 SOP 用现有版本测试 |
| 7 | **B1 节点压力**：所有相关工作要在 B1 之前关闭 | 认证延期 | 节前完成清单+SOP，禅道闭环跟踪 |
