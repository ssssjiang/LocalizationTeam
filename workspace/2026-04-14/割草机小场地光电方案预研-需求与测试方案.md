# 割草机小场地光电方案预研 — 需求分析与测试方案

> 整理日期：2026-04-14
> 来源：项目群讨论（树林 / 曹丽娜 / 宋姝）

---

## 一、背景与目标

### 1.1 业务背景


| 项目   | 内容                                    |
| ---- | ------------------------------------- |
| 应用场景 | 割草机，**小面积场地**（非大型开放环境）                |
| 核心驱动 | 用便宜的面阵固态激光雷达模组，在小场地实现建图定位，效果优于纯双目视觉方案 |
| 成本约束 | 面阵激光模组价格**对标双目相机模组**                  |
| 竞品参考 | MOVA（纯双目方案），作为效果基线                    |


### 1.2 核心目标

1. 验证面阵固态激光雷达模组在小场地的建图定位能力
2. 与纯双目 SLAM 方案横向对比，量化优劣
3. 评估单目 VIO 在激光观测退化时的兜底能力
4. 摸清竞品（MOVA）纯双目方案在典型场景下的实际表现与边界

## 二、硬件配置

### 2.1 自研面阵激光模组

```
面阵固态激光雷达
  └─ 水平 FOV 120°
  └─ 输出：点云

单目 RGB 相机（rolling shutter）
  └─ 输出：图像帧

IMU
  └─ 输出：惯性数据
```

> **注意**：点云与图像像素**不是一一对应关系**（分辨率、帧率独立）。
> 两者之间需做外参标定，但点云密度与图像分辨率无关。
> 传感器组合本质为：**LiDAR + Mono + IMU**，不是传统像素对齐的 RGBD。

### 2.2 竞品机器

- 纯双目相机方案（MOVA）
- 使用方提供机器，在相同场地运行进行横向对比

## 三、自研方案验证计划

### 3.1 数据采集内容


| 数据类型   | 来源              | 用途            |
| ------ | --------------- | ------------- |
| 点云     | 面阵 LiDAR        | SLAM 主观测      |
| 单目图像   | RGB 相机（rolling） | 视觉辅助 / VIO    |
| IMU 数据 | IMU             | 运动补偿 / VIO 积分 |


### 3.2 验证目标


| 验证项                     | 说明                   | 优先级 |
| ----------------------- | -------------------- | --- |
| 主链路：LiDAR+Mono+IMU SLAM | 建图精度、定位稳定性           | P0  |
| 激光退化兜底：单目 VIO           | 激光观测质量下降时 VIO 能否维持定位 | P1  |
| 跨区域建图                   | A 区建完后继续建 B 区及连接通道   | P1  |
| 回桩精度                    | 定位精度的强验证             | P1  |


### 3.3 风险点


| 风险                   | 说明                               |
| -------------------- | -------------------------------- |
| 户外草地点云质量             | 草地近地面低反射、遮挡，LiDAR 特征稀少           |
| Rolling shutter + 振动 | 割草机振动可能导致图像模糊，影响视觉跟踪             |
| 激光/视觉同时退化            | 无纹理草地 + 特征稀疏点云，两个传感器可能在同一场景下同时失效 |
| 低光照下单目 VIO           | 黄昏/日落后相机曝光不足，VIO 兜底能力存疑          |

## 四、竞品（纯双目）测试方案

> 来源：曹丽娜 14:21 整理

### 4.1 测试场景

#### 基础场景

- 每个场地：**建图 1 次，割草 2 次**
- 割草时间要求：
  - 第 1 次：**光照条件好**（白天）
  - 第 2 次：**黄昏或日落后**（低光照）
- 对桩位置**无特定要求**

#### 额外场景（跨区域）

```
布局：
  [场地 A] ── 窄通道 ── [场地 B]

流程：
  Step 1：建完 A 区域地图
  Step 2：继续建立 B 区域 + 通道地图
  Step 3：机器在 AB 区域完成完整割草任务

测试次数：共 2 次
  - 光照好的时候 1 次
  - 光照不好的时候 1 次
```

### 4.2 输出要求


| 输出物    | 说明                      |
| ------ | ----------------------- |
| 监控视频   | 建图过程 + 割草过程全程录像         |
| App 录屏 | 建图界面 + 割草界面的屏幕录制        |
| 割草记录截图 | 最终割草完成状态截图              |
| 异常记录   | 任何问题须记录，例如：无法回桩、无法通过通道等 |
