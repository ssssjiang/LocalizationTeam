# SLAM暂停

# 1. 核心需求

不需要SLAM的时候暂停，省电、省CPU、省内存，降低散热

# 2. 暂停发生的情况

1. 按下stop键

2. 报错

3. 在桩上

4. 桩外待命

5. 升级

# 3. 暂停（pause）和之后的恢复（resume）

1. 参考扫地机：pause后，轮子不动就不给导航更新定位，来odom消息的时候（这时候其实是静止），始终发给导航暂停前的pose，导航在整个pause期间要监听抬起信号、gyro超过5度信号、odom移动信号（推荐）

   1. 如果pause期间slam收到Gyro\odom即使有发生移动的传感器信息，slam不dispatch

   2. 如果pause期间slam收到激光雷达、RTK等定位信息，slam不dispatch

      1. 理论上不会发生，pause的时候，定位传感器应该为了省电而关闭

2. 不参考扫地机：先resume定位，等定位的response，再启动导航（不推荐）

3. 暂停期间不检测传感器，不进行重定位

# 4. 暂停期间发生搬动、gyro超过5度信号、odom移动

暂停指令状态机发

导航记录“曾发生搬动”这个状态

resume状态机发，导航发现resume之后，通知slam开始重定位，并启动重定位动作

# 5. 与暂停相关的SLAM改动

1. odom和IMU传感器更新量小于一定阈值，不递推融合SLAM的状态

2. 轮子停（导航给驱动发零速指令 ）2s后帮看一下

   1. 用2s的滑窗滤波，进行角速度偏置的估计

   2. 两个判断结果：

      * 完全停止

      gyro (fabs(均差) <= 0.03 && fabs(峰值) <= 0.1)

      acc (峰峰值<0.35)

      * 稳态

      gyro (fabs(均差) <= 0.04 && fabs(峰值) <= 0.05)

   3. 满足 完全停止 条件

      更新offset

   4. 满足 稳态 条件

   角速度强制设为0，只使用acc进行更新

   持续2s后

   acc使用初始对准方法计算roll和pitch，yaw保持不变
