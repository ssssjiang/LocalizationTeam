## 1. 整体时序

```
退桩距离达到 1m
    └─► 立即发送「建通道消息」（不等待后续阶段）
            │
            ▼
    继续退桩至 S0（约 1m）
            │
      SLAM 初始化成功? ──否──► 继续退至 S1（再退 1m）
            │                         │
           是                   SLAM 初始化成功?
            │                    是 ◄──┘   否
            ▼                    │          │
    导航反算出桩通道         正常流程      用户手摇建图
            │
            ▼
    VSLAM 建图（后台，约 1min 优化）
            │
    建图成功? ──否──► 定位不发轨迹
            │         导航标记通道类型：RTK 通道
           是
            │
            ▼
    定位发送「优化后的通道轨迹」
    （出通道后约 1min 延迟）
            │
            ▼
    导航收到消息后更新通道坐标
    导航标记通道类型：视觉通道
            │
            ▼
    默认存储出桩 + 回桩两条通道
    App 显示：两条相似则仅展示一条
```

> 来源：`inbox/通道建图/通道视觉建图接口对齐 - 4.7.md`；`inbox/通道建图/RTK阴影区出桩VSLAM建图定位-导航接口.md`（3月25日更新）

### 2.2 退桩 2m 后逻辑（改动）

**改动前**：退桩 2m RTK 未初始化成功 → 重定位失败后直接报错

**改动后**：

```
退桩 2m（S1），RTK 未初始化成功
    └─► 不报定位失败；插件切换至用户遥控模式
            └─► 用户遥控机器人前往信号好的区域
                    └─► RTK 初始化成功
                            └─► 导航根据 SLAM 初始化结果反算出桩通道
                                    └─► 视觉建图优化结束，发送轨迹
                                            └─► 通道轨迹修改为优化后的轨迹
```

附：**取消**二阶段退桩失败之后的重定位逻辑

> 来源：`inbox/通道建图/通道视觉建图接口对齐 - 4.7.md` §退桩 2m 后逻辑修改；`inbox/通道建图/【RTK 款】阴影区域支持VSLAM 建图.md` §1.2

### 2.5 出桩 / 回桩双通道存储与显示去重

- SLAM/导航**默认存储出桩和回桩两条通道**
- SLAM 会发送出桩、上桩**两条**优化后的通道轨迹
- App 显示时：若两条通道**相似，仅展示一条**

### 2.6 通道编辑通知 — 当前版本接口兼容方案

- 当通道被编辑（**通道 id**、**任一端区域 id** 有变更），导航向定位发送**删除该通道**的消息
- **不新增接口**，保证本次发版稳定

### 2.7 割草退桩逻辑

1. 导航发送桩点坐标给融合，融合从桩点递推，发送（伪）RTK 坐标系下 pose 给导航
2. 导航退桩 2m 后，RTK 未初始化成功，导航走**三角型重定位轨迹**，但三角型的**一条边贴近通道**
3. 割草的出桩通道：在 SLAM 初始化成功之前，导航需要根据 **RTK 系下的出桩通道**走

> 来源：`inbox/通道建图/通道视觉建图接口对齐 - 4.7.md` §割草退桩逻辑；`inbox/通道建图/RTK阴影区出桩VSLAM建图定位-导航接口.md` 3月25日更新第 4 条

## 4. 通道截断策略（导航）

当新建区域截断已有通道时，导航按如下规则处理通道 id：

**CASE 1：新建区域仅截断已有充电桩通道**

- 假设 region2 与充电桩通道 id=1，新建 region3 截断通道 1
- 通道 1 分为两段：region3 与充电桩的通道**继承 id=1**；region2 与 region3 的通道取**最小未使用 id**

**CASE 2：新建区域截断已有充电桩通道且与已有区域有交叠**

- 新建 region3 继承原有 region2 的通道；通道 id=1 被截断（删除）一部分；region2 不再直接与充电桩相连
- 若 region3 与 region2 融合：新 region 继承最小区域 id，通道 id 处理方式与重叠处理一致

**CASE 3：新建区域的充电桩通道被已有区域截断**

- 新建 region3 的充电桩通道经过 region2，则新通道被截断
- region2 将有两条充电桩通道；region2 与 region3 将有一条普通区域通道

> 来源：`inbox/通道建图/通道截断策略——导航/通道截断策略——导航.md`
