# okvis&maplab对比

|    |                                                   | okvis                                                              | maplab                                                                                                   | okvis+maplab一起用？                         |
| -- | ------------------------------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- | ---------------------------------------- |
|    | 优势                                                | 功能比较完善，研发周期较快                                                      | 建图优化方面精度高现成的和老地图关联的机制                                                                                    |                                          |
|    | 缺陷                                                | 建图优化方面精度低维护了两个全局图，内存占用高（额外的fullGraph单纯用来做优化，优化完成后同步到realtimeGraph） | 建图过程中的重定位功能需要开发                                                                                          |                                          |
|    | 精度                                                | 低                                                                  | **高**                                                                                                    | okvis的回环优化or**maplab**全图优化               |
| 建图 | **完成建图过程中的重定位功能。**                                | **有**精度、成功率待测基于kf找，理论上成功率会更高一些                                     | 无未优化的地图直接找回环，成功率降低80%有提升空间[ 地图优化前后精度及重定位效果](https://roborock.feishu.cn/wiki/A3n1w39UjiOfmUk8plIcKYohnFf) | 有（okvis多会话or **maplab**未优化图重定位）          |
|    | 建图过程中有足够高的精度                                      | **有**通过回环优化提升精度会有姿态突变                                              | 无                                                                                                        | 有（okvis回环）                               |
|    | 完成重定位成功后，将后续VIO定位数据附加到已有的建图数据上的功能。                | 无，开发量相同                                                            | 无，开发量相同                                                                                                  | 待开发（okvis多会话）                            |
| 优化 | 完成带中断轨迹的地图优化功能。（预期风险不大）                           | 无，需要额外开发                                                           | 无，需要额外开发                                                                                                 | 待开发或者舍弃（okvis fullgraph 全局BA）            |
|    | 建图优化时间足够快（目标为板端400平2\~3分钟）                        | **有**                                                              | 无目前5分钟                                                                                                   | okvis的回环优化、全局BA、姿态图优化**maplab**全图优化      |
| 定位 | 重定位多帧校验                                           | 无，开发量相同                                                            | 无，开发量相同                                                                                                  | 待开发                                      |
|    | 重新割草时必须与持续与老地图关联（否则每次割草自己的地图精度再高，也不一定和老地图对齐，导致出界） | **有**                                                              | **有**                                                                                                    | 有&#xA;（okvis与已存地图回环 or **maplab**重定位加观测） |

1. okvis的建图功能打通

   1. 多会话功能

      1. 功能点：对于多个地图，okvis原版算法输出当前帧对于多个地图的坐标系变换

      2. 使用方式：

         1. 可以用于建图过程中的重定位

         2. 或者还有maplab时用maplab的图，多会话功能这部分废弃

   2. 回环检测&优化

      1. 功能点：

         1. 在realtimeGraph上找回环

         2. 窗口搬动

         3. 在fullGraph上创建约束，发给后台线程优化

      2. 可以应用

         1. 好处：提高建图过程中的精度
