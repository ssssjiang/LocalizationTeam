# 窄通道建图接口讨论 2026年3月17日

> 会议主题：窄通道建图接口讨论
> 会议时间：2026年3月17日

## 讨论结论

### 1. 坐标转换

1. vslam 地图坐标系复用 vio 坐标系，所以 vslam 地图和 rtk 地图的对齐可以复用 vio 和 rtk 对齐的结果
2. 进入通道（收到导航"进入通道"消息）的时候，融合强行进行一次对齐，这个对齐的结果作为 vslam 地图和 rtk 地图的坐标转换关系

### 2. vslam 和融合的接口

一共两个 flag：

1. **vio？vslam**（v 是否重定位成功）
  - 进入通道连续重定位成功 5 次（vio 和 vslam 一致）改为 vslam 状态
2. **定位？建图**（vslam 是否需要融合给出对齐的坐标转换关系）

vslam 需要告诉融合现在是 vslam 定位状态还是 vslam 建图状态：

- **如果是建图**：
  - 需要融合发给 vslam vio 到 RTK 的坐标转换关系
  - 融合工作在 RTK 或 vio 模式下
- **如果是定位**：
  - vslam 发 RTK 系下的坐标（vslam 自己转换），需要带 flag（标记 vslam）
  - 融合工作在 RTK 或 vio（满足使用视觉的条件）或 vslam（视觉重定位成功）状态下
  - 每次出通道 vslam 都发通道和 RTK 的转换关系
    - 如果一直是 vio flag（vslam 全程没有重定位成功），也强制发这个转换关系，融合自己决定是否使用
- **如果是 vio**：
  - vio 发 vio 坐标系下的坐标，需要带 flag（标记 vio）

### 3. 通道内 vslam 定位的出通道行为

通道内如果是 vslam 定位：

- 出通道后如果是 RTK 固定解，融合满足切换为 RTK 的条件，呈给切换为 RTK 定位，但很快又有浮点解，切换为 vio 定位，此时需要做一次对齐。对齐的时候如果通道 + 通道外 RTK 长度不足 5m，改用 vslam 地图坐标对齐
- 通道内如果一直没有 vslam 重定位成功，使用 vio 定位，出通道后直接使用 vio 对齐

### 5. 视觉优化后的通道发布方式

1. 2D 轨迹（不含姿态）
2. 简单 PGO？还有没有其他方案？
3. vslam 直接发，不需要经过融合
4. 轨迹点间隔 5cm 以内，间隔过小就滤波，间隔过大就插值
5. 通道 2D 坐标通过广播消息发出来
  - 通过 IPC 消息发可能有限制
  - 40m / 0.05m = 800 个点 = 1600 个 float

### 6. merge vslam 地图功能

- merge vslam 地图功能在 HF2.0 关闭
- 正反向通道默认两张 vslam 地图

### 7. 上桩/出桩路径

用户遥控的上桩路径和出桩路径无法"合并"为一条，按照两条路径编号处理
