# 03-通道建图 — 定位组算法细节

> 来源：[ 窄通道建图接口讨论 2026年3月17日](https://roborock.feishu.cn/docx/CDbGdWDxzonBigxP1lEcPOipnTf)

## 1. 建图策略

### 1.1 SLAM 初始化流程

* 退桩期间持续进行 **slam 局部定位**

* SLAM 初始化成功后，将 vslam **局部坐标系发给导航**，由导航做局部坐标到世界坐标系的转换

* S0 点（退桩 1m）初始化成功：导航反算出桩通道

* S1 点（退桩 2m）仍未成功：用户手摇继续建图

> 来源：[ RTK阴影区出桩VSLAM建图定位-导航接口](https://roborock.feishu.cn/wiki/EsnIwWPMVi2ULgk3YMkc2woVnyg)

### 1.2 建图结果处理

* 发布频率：**7Hz** 建图轨迹

* 建图成功：发送**优化后的通道轨迹**

* 建图失败：**不发轨迹、不报错**，由导航标记为 RTK 通道

***

## 2. 通道轨迹发布

* 格式：**2D 轨迹（不含姿态）**

* 发布方：**vslam 直接发，不经过融合**

* 发布方式：广播消息（IPC 消息有大小限制，不适用）

* 轨迹点密度：间隔 **≤ 5cm**；间隔过小则滤波，过大则插值

* 数据量估算：40m / 0.05m = **800 个点 = 1600 个 float**

* 方案选型：插值到 vio 频率，fix vslam kf，添加 vio 相对位姿约束，做  PGO 优化；

***

## 3. 定位策略

### 3.1 **vio / vslam** flag

* &#x20;TGV pose filter 收敛之后，重定位成功，切换为 vslam 状态；

* vslam pose 为世界系为 RTK 全局世界系；

* vio pose 在 vio 从 0 启动的世界系下；

### 3.2 出通道后的定位切换行为

* 在离开通道前， vslam 发布 TRV；（RTK 全局世界系到 vio 世界系的变换）

* 出通道后遇 RTK 固定解 → 融合切换为 RTK → 很快变浮点解 （或一直为浮点解）→ 切换回 vio → **需做一次对齐**

  * 对齐时，若通道 + 通道外 RTK 长度 **< 5m**，改用 vslam 发布的 TRV；

* 通道内始终未重定位成功（全程 vio flag）→ 出通道后直接使用 **vio 对齐？**

## 4. 轨迹多帧对齐方案

> 来源：[ 视觉通道建图-轨迹多帧对齐](https://roborock.feishu.cn/docx/CGNTd9p2YoZ0TCxUPPmcO4kTndf)

### 4.1 背景与问题

历史版本为，进通道后，定位模式下，vslam 发送请求，融合发送 TRti，vslam 计算 `TRG` = TRti \* TGti.inv （视觉局部坐标系到世界坐标系的变换）由**单帧计算**得到，实测对齐误差偏大。

### 4.2 新方案

采用**多帧对齐**方式计算 `TRV`（[ 视觉通道建图-轨迹多帧对齐](https://roborock.feishu.cn/docx/CGNTd9p2YoZ0TCxUPPmcO4kTndf) 文档写作 **`Twv`**，表达的都是 RTK 全局世界系到 VIO 世界系的变换） 。

### 4.3 额外开发内容

**融合模块（当前版本）**：

* 主动 query 时，发送**多帧对齐的 `Twv`**，同时发送对齐数据的**起止时间；**

**视觉模块（当前版本）**：

* 接收并记录 RTK 固定解、VIO 位姿、vslam 建图位姿；

* 根据通道内 RTK 固定解情况，按流程图描述计算 `TWG`，用于建图位姿转换；

**第二版（可延后）**：

* 融合模块：创建通道中时，发送经过过滤的 RTK 固定解（含时间戳+位置）；

* 视觉模块：RTK 数据的接收和对齐；

### 4.4 排期

* **4/13 前**：融合、视觉各自完成第一版私分支开发（基于 vio 串联 RTK 与 vslam）

* **4/13**：开始联调

## 5. 已知限制与风险

| 限制/风险        | 说明                                    | 状态          |
| ------------ | ------------------------------------- | ----------- |
| 通道长度上限 40m   | 超过 40m 建图失败                           | 当前版本约束      |
| 视觉频繁保图 IO 卡顿 | 建图过程中频繁写图可能造成 IO 卡顿                   | 风险待验证       |
| 视觉建图优化时间较长   | 约 1min，掉电/OOM/Core 等异常可能打断优化          | 需确认中断后的恢复行为 |
| 进/出通道姿态跳变    | 视觉地图优化后，进/出通道瞬间可能出现定位飘（约 1m），信号不好时更明显 | 已知问题        |
