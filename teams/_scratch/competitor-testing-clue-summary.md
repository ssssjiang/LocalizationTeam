## 1. 文档 1: `inbox/0418新增/26.1-德国竞品测试/26.1-德国竞品测试.md` (1420 行)

**文档侧重**：德国实地测试，以**单机分章节**的方式列测试项（每个机器一节，标题统一从开箱→建图→定位→避障→刹车→边界等）。模板项很多但很多机器没填完整。
**测试地点**：Neckarstraße 20A Altes Forsthaus, Dormagen（民宿场地，含鼹鼠打洞场景）。

### 涉及竞品 / 产品

- **九号 X3**：填得最详细（RTK 盖锅子定位专项是高价值内容）
- **九号 i108**：模板填了大半，定位相关信息较少
- **Sunseeker X7**：边界/禁区/避障为主，定位信息少
- **松灵 Luba mini AWD LiDAR**：填得详细（建图偏差专项 + 重定位动作 + nRTK 失效场景）
- **松灵 Yuka（nRTK 专项）**：明确是 nRTK ↔ RTK 切换专项
- **Mova viax300**：仅"开箱"

### 涉及方向（定位视角）

- **融合 (RTK 主)** 占绝对大头：九号 X3、i108、Sunseeker X7、Yuka，全部围绕 RTK / nRTK 行为
- **激光** 主要是松灵 Luba（360 激光）
- **视觉** 主要体现在：九号 X3 建图后的"视觉深度学习"(~75min)、Luba 的双目沿边、AI 自主建图

### 定位/建图相关测试项（粗列表）

**A. RTK 信号差行为专项（九号 X3，全文最大亮点）**

- 局部建图 RTK 盖锅子（1min 内不报错；轮子空转地图飘）
- 完整建图全程盖锅子（信号差地方偏移大；信号好处隔一段跳变校准回来 → 疑似用机身 GPS 1.5m 级单点定位兜底）
- 建图盖锅子 / 割草不盖锅子（地图实际不能用）
- 割草中盖锅子（地图飘 4-5 个机身后进脱困）
- 拿走锅子定位闪回正确
- 信号好时提起轮子空转，定位不会飘

**B. nRTK / RTK 切换行为（YUKA 专项）**

- nRTK ↔ RTK 互切是否需要重建图（不需要）
- nRTK 转 RTK 后 UI 出现 RTK 标识
- YUKA 首次进入界面默认非 nRTK，需手动切
- YUKA nRTK 信号源可选但不自动切换
- YUKA 三种定位方式
- YUKA nRTK 模式下关闭 4G & Wi-Fi 后仍可割草约 23min，再报 RTK 基站未就绪

**C. 重定位 / 续建（出现在多机）**

- 九号 X3：支持重定位、续建（仅一句确认）
- Luba：关后台后可续建；建图中搬动机器→建图失败
- Luba：3D 实景图效果展示（涉及 RGB 重建可视化）

**D. 出界 / 拉出边界（与定位边界判定相关）**

- 九号 X3：拉出 3m 内会回到界内继续割草，超 4m 先重定位再报错

**E. 建图相关（与建图引擎/拓扑相关，影响定位）**

- 九号建图无大小限制
- 地图重叠 / 大图包小图能保存且可手动合并
- 通道穿越多个草皮、孤岛标志
- 通道、禁区、新建草坪交叉拓扑（6 项左右）
- Luba 建图末端定位偏差导致无法闭合（需顶围栏打滑校准）
- Luba 建通道时定位偏差导致建通道失败

**F. AI 自主建图 / 建图后视觉学习**

- 九号 X3 建图后视觉深度学习（22% → 100% 用 75min，1800㎡）
- Luba AI 建图（口述：边界模糊就转手动）
- 九号 i108 建图后桩上视觉深度学习（显示 125min）

### 涉及场景 / 专题

- RTK 盲区（盖锅子人造）
- nRTK 4G/Wi-Fi 切换 + 4G 中断
- 鼹鼠打洞 / 卡坑（影响里程计/打滑判定）
- 边界与禁区严格性（多家都不严格、九号 X3 / Sunseeker X7 / 白马都"画了禁区也掉")
- 通道：穿越多草坪、与禁区/草坪重叠

### 文档侧重判断

**实地体验式 + 模板化测试项混合**。模板项很多但留空率高；定位类高价值内容主要集中在"九号 X3 RTK 盖锅子专项"和"YUKA nRTK 专项"和"Luba 建图偏差现象"三块。

### 定位/建图相关测试项（粗列表）

**A. 重定位 / 续建（最核心的定位类章节，§2.4）**

- 九号 1+1 RTK (i206) — 5 个细分场景：
  - RTK 好 + 搬 1m + 朝向一致 → 实时更新，地图可用
  - RTK 好 + 转 3 圈 + 朝向一致 → 实时更新，搬运段被优化掉
  - RTK 好 + 转 3 圈 + 朝向偏 90° → 朝向稍偏，搬运轨迹优化偏差较大
  - RTK 阴影 + 搬 1m + 朝向一致 → 定位不更新；视觉优化后地图整体偏 1m 不可用
  - RTK 阴影 + 搬 1m + 朝向偏 90° → 同上
- Luba nRTK：搬运/拖拽轨迹与朝向实时更新；轨迹闭合则报失败（不支持 8 字）

**B. AI 自主建图（§2.7）**

- 九号 AI 建图：沿边探索 + 弓字内部探索；遥控到草坪内方可点亮"标记"按钮（识别 1-2m，实测有时 3-4m 才识别）
- 异常处理：悬崖翻车 → 主动重定位；重定位识别在已建草坪外则无法续建
- 切状态：建图中切割草/回充仍可手动续建

**C. 脏污影响（感知 → 定位，§2.2 + §8）**

- 建图开始时 / 过程中盖摄像头
- 雨水、泥点
- i206 单目下定位信号正常 + 雨/咖啡/泥脏污：可继续割但效率低；严重脏污 5min 后报错停机

**D. RTK 管理（§3）**

- RTK 热力图（生成 / 更新逻辑，问题列表，未填实验）
- 充电桩位置安装推荐（基于热力图）

**E. 防盗 / 搬运告警（与定位有关，§3）**

- 防盗围栏 5-50m / 10-50m，依赖 GPS（信号好时误差 2-5m）
- 拔 RTK 后从桩搬出 → 长时间未触发防盗告警，App 显示位置未更新（说明依赖 RTK 位置）

**F. 出界 / 边界判定（§15.出界）**

- Yuka Mini：1.2m 即报出界；返回到 0.4-0.6m 内才能启动
- 九号激光 i2：慢推 3m 报错；抬高后轮拉出 3m 后做重定位再正常回充
- 九号 RTK 款：同激光行为；割草中搬至室内会一直挣扎走不停

**G. 寻桩 / 充电桩位移（§11，与定位地图一致性相关）**

- 左右挪 1.5m 内能找回，2m 悬
- 上桩成功后退桩报"充电桩位置变化"

**H. 通道（§5）**

- 多通道策略（未填实测）
- 窄通道（链接到飞书 wiki，未填实测）
- 草坪内狭窄区域

### 涉及场景 / 专题

- **重定位 × 搬运姿态（朝向一致/偏 90°）** ← 高价值

### 涉及方向（定位视角口述要点摘录）

**融合 (RTK)：**

- 九号 X3 RTK 盖锅子：信号好就跳变校准（GPS 1.5m 级别用于兜底），地图整体偏 1-2m
- 单点 GPS 精度：定位组曹丽娜口述"最差能到 4.5m，均值 2-3m"；李圳"全频单点 1.5m 内"
- Yuka nRTK 4G 强绑定：4G 过期 → nRTK 失效；不动态切换 4G/Wi-Fi

**激光：**

- 松灵 Luba 120° 面阵激光：在开阔场景定位偏差 → 建图回到原点偏差 2 个机身，闭合不了
- 同款机器选配 RTK：只有特征点充足时面阵激光定位才好

**视觉：**

- 九号 X3 建图后视觉深度学习（沿边一圈，桩上学习 75min，1800㎡）；学习后说能更贴边但实测没看到明显贴边提升
- Luba AI 建图：边界模糊就让人接管（速度慢、不可靠）
- Luba 单目 + 双目 vs 单目避障：对远处物体（如人后拉着机器）单目避障表现"魔幻"（杆能识别，但人推机器扭矩检测不出）

**其它跨方向（与定位间接相关）：**

- 视觉脏污：松灵遮单目 → 仍开刀盘走，识别定位差时关刀报"定位信号弱"
- 围困：Luba 围困场景下不报错，预规划路线持续推进
- 充电桩位移：松灵地图不更新桩位置，但红外能扫到就能回桩
- 定位信号好/差对回桩 / 防盗的影响

### 涉及场景 / 专题（与定位有关的）

- **RTK 盖锅子模拟信号差**（九号 X3）
- **nRTK 4G 中断**（YUKA）

- **单点 GPS 兜底跳变**（理论讨论 + X3 现象）
- **视觉脏污 → 定位失效报错**（Luba）
- **围困 / 异常封闭区域定位**（Luba）
