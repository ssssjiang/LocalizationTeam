# 3. 重定位

## 3.1 窄通道用户抱起

|         | 放在窄通道内                                            | 放在窄通道外                               |
| ------- | ------------------------------------------------- | ------------------------------------ |
| 触发重定位   | 激光割草机窄通道                                          | 重定位正常成功、失败（低概率）、假成功（低概率）             |
| 触发减少重定位 | 跟窄通道内定位接近，&#xA;但是没有odom约束，很容易就失败了&#xA;但是要想办法防止假成功 | 由于窄通道内的部分，导致定位失败或假成功。&#xA;要想办法防止假成功。 |

## 3.2 窄通道内重定位

* 累积局部地图&#x20;

  * 问题：&#x20;

    * 当前不使用2.5m范围内的点&#x20;

      * 局部地图会花，无法成功累积局部地图

    * 如果使用2.5m范围内的点&#x20;

      * 会有操作人员导致的杂点，可能导致局部图与全局图重定位失败或假成功&#x20;

        * 待验证

  * 初步解决方案：&#x20;

    * 累积局部地图的时候分成两张局部地图&#x20;

      1. 用来定位的局部地图

      * 只保证局部地图内定位准确，可以构建局部地图

      * 使用所有点云

      2. 用来重定位的局部地图

      * 用来跟全局地图匹配的局部地图

      * 2.5m范围内的点只使用地面点&#x20;

        * 如何获取地面点？（待验证）&#x20;

          * 高度滤波

          * 法向量滤波

      * 为什么要保留2.5m范围内的地面点？&#x20;

        * 在特征匹配（x\y\yaw）这一步之后，需要使用局部地图的地面点和全局地图地面点进行z\pitch\roll的匹配

          * 如果没有地面点，局部地图和全局地图匹配的时候，会发生翘起，导致评分不过阈值，被判定为失败

* ~~扩展地图重定位~~（感觉应该废弃） ------

  * 如果当前重定位方案局部地图构建的时候，地图花，那这个方案也有一样的问题，构建地图也没法和扩展地图进行拼接

  * 如果当前重定位方案局部地图跟全局地图匹配的时候假成功，那这个方案在扩展的时候也有假成功情况

  * 所以这个方案不能解决任何实际问题，应该暂停
