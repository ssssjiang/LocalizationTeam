- **双目填补提议**（丛一鸣）：被李哲否定，认为技术路线不一致、较费劲
- **参考图**：![空洞示例](images/fastlivo_panorama_spatial_hole_example.jpeg)
- **建议确认人**：李哲（2026-04-14 已接 action）
- **优先级**：高（坡道空洞当前已可复现）

### Q-008 阶段二机器实时位置显示的坐标系同步

- **问题**：导航定位算法坐标系 与 实景地图显示坐标系 是独立两套，阶段二若要显示机器实时位置需要对齐；同步方案未设计
- **建议确认人**：刘宏伟
- **优先级**：低（阶段二，当前不需要）

### Q-009 机器端 RAM 容量是否够用

### Q-010 多次割草点云拼接质量

- **问题**：多次出桩/回桩产生多块点云，各块坐标系是否对齐、拼接后是否有可见重影/分层，尚无实测数据

- **建议确认人**：刘宏伟（采数据）+ 李哲（评估结果）
- **优先级**：高（当前最重要的未知项）

### Q-011 导航百分百覆盖率接口

- **问题**：D-011 方案中以"区域覆盖100%"为地图更新终止条件，需要导航模块提供对应接口；接口是否存在、格式未确认
- **背景**：丛一鸣提出，导航知道区域是否百分百覆盖，需要对接
- **建议确认人**：导航团队
- **优先级**：高（影响地图更新策略实现）

### Q-012 ISP 参数对彩色效果的影响边界

- **问题**：若后期想调整点云彩色效果，可能涉及相机 ISP 参数调整；ISP 会影响其他感知识别模块，不能随意修改
- **背景**：李哲 2026-04-14 提出风险，当前彩色效果（路面/草地）丛一鸣认为 ok
- **建议确认人**：感知团队（ISP 影响范围评估）
- **优先级**：低（当前彩色效果可接受，暂不需要改）

### Q-013 地面补全 `sensor_ground_offset` 参数语义需澄清

- **问题**：D-016 文档中该参数语义存在歧义：
  - 方法描述与公式 `ground_z = pose.z - sensor_ground_offset` 把它当作"传感器到地面的高度偏置"
  - 参数表却写"地面高度估计时搜索已有地面点的半径"
  - 推测应是两个独立参数（高度偏置 + 搜索半径），文档合并写错
- **影响**：参数表是后续调参/移交的唯一真源，含义不明会直接影响他人复现
- **建议确认人**：D-016 方案作者
- **优先级**：高（影响代码与文档一致性）

### Q-014 地面空洞补全的量化验收指标

- **问题**：D-016 当前结论"补全后结构与色彩复原效果较好"为定性描述，缺少：
  - 补全点占比、补全前后空洞栅格比例
  - 颜色一致性 / 上错色率
  - bad case 统计（哪些场景下会失败：重纹理地面、阴影、湿地反光等）
  - 对融合定位 / FAST-LIVO2 后续重定位是否有副作用
- **建议确认人**：刘宏伟 + D-016 方案作者
- **优先级**：高（定性结论不足以作为发版依据）

### Q-015 地面补全方案是否需要"墙面开关"

- **问题**：当前方案明确"不补墙面"，理由是颜色/纹理差异大风险高；但室内白墙、色块均匀的外墙等弱纹理场景可能可以受益。是否设计开关或自适应判定，待评估
- **建议确认人**：D-016 方案作者 + 李哲
- **优先级**：低（当前形态外场割草机为主，墙面非主诉求）

### Q-016 拼接重合度阈值未量化 + 失败片段处理未定

- **问题**：D-017 的"重合度 = 匹配点 / 大图点数"判定缺核心要素：
  - 重合度阈值是多少？（无阈值 = 无法复现、无法 review）
  - 重合度分母用"大图点数"反直觉：大图越大分母越膨胀，**新片段越往后越难判为重合**；建议改用 `匹配点 / min(候选点数, 大图局部 ROI 点数)`
  - 重合度未达标的片段如何处理？丢弃 / 独立保留 / 进入"待二轮"队列等大图扩张后重试？
- **建议确认人**：D-017 方案作者
- **优先级**：高（影响算法可复现与拼接覆盖率）

### Q-017 实景拼接（D-017）与地面空洞补全（D-016）的执行顺序

- **问题**：两者都在"回桩后离线后处理"阶段运行，顺序未定：
  - **先补再拼**：每个片段各自补地面 → 拼接时补全点参与重合度计算 → 同一块地面可能被补两份，重叠区颜色冲突
  - **先拼再补**：拼好大图再统一补地面 → 需要"全局轨迹"，每个片段轨迹要在拼接时一并携带与合并
- **影响**：两个 V1.0 同时上线会冲突，必须明确顺序与中间产物口径
- **建议确认人**：D-016 / D-017 方案作者 + 李哲
- **优先级**：高（发版前必须明确）

### Q-018 全景地图 index 状态文件的异常恢复

- **问题**：D-017 落盘机制依赖 `/mnt/data/rockrobo/index` 单调自增，但缺兜底：
  - index 文件丢失 / 损坏怎么办？
  - 升级 / 重置后 index 不连续，扫描目录如何识别有效片段？
  - 建议：进程启动时把 index 校准为「目录现存最大编号 + 1」
- **关联**：`/dev/shm/temp_X.pcd` 在 `FSM_CHARGE` 之前异常断电会丢失，**无 checkpoint 机制**，是否需要定期 flush 也属同类问题
- **建议确认人**：D-017 方案作者
- **优先级**：中
