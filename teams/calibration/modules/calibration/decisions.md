## D-004 标定结果存储策略：EEPROM 主存 + JSON 双备份【已定案】

**背景**
标定结果需要在机器重启后持久化，且需要应对 EEPROM 读取失败的情况。

**选择**

- 主存：EEPROM（驱动层接口 `rr_write_dual_cam_calib_result` 等）
- 备份：`/mnt/reserve/dualcamera_aftersale_calibration.json`（售后标定结果）和 `dualcamera_calibration.json`（出厂标定结果）

**读取优先级**

1. EEPROM（任意接口失败则 fallback）
2. `dualcamera_aftersale_calibration.json`（有则使用）
3. `dualcamera_calibration.json`（兜底）
4. 全部失败则报错

**dualcamera_calibration.json 中 Tcrcl 字段保存要求**

Tcrcl JSON保存格式

**原因**

- EEPROM 读写速度快，是运行时标准路径
- JSON 双备份确保出厂参数和售后更新参数均可恢复
- 售后参数优先于出厂参数，保证最新标定结果生效

**来源** | 售后-双目行差检查与行差标定方案（§ 机器启动后的参数读取流程）

## D-005 售后标定与双目行差标定合并执行【已定案】

**背景**
双目模组老化会导致左右目行差（行方向对齐偏差）增大，影响定位精度。行差标定需要类似的运动激励，与售后外参标定流程高度重叠。

**选择**
正常的售后标定流程中，**同时执行双目行差标定**。如果外参变化不大则不输出新外参，仅在变化超阈值时输出。

**原因**

- 合并执行复用同一套采集流程，不增加用户操作步骤
- 行差结果写入 EEPROM 预留空间，与外参写入区域分离，互不干扰

**来源** | 售后-双目行差检查与行差标定方案（§ 1.整体流程）

## D-006 侧目相机参数：鱼眼模型 + 三段 JSON 格式【已定案】

**背景**
侧目相机为鱼眼镜头（分辨率 640×544），需定义从出厂到标定完成的参数流转格式。

**选择**
三段式 JSON：

1. **厂商 EEPROM 输入格式**：仅有内参和畸变（无外参、无 position 字段）
2. **标定程序输出格式**：增加外参（旋转矩阵 + 平移）和 position（left/right）
3. **EEPROM readback 格式**：与输出格式一致，用于回读验证

畸变模型：fisheye（k1～k4）。

**原因**

- 三段格式对应标定流程的三个阶段，职责边界清晰
- 未标定时外参发布旋转为单位阵、平移为 0，保证未标定情况下系统可运行

**来源** | 侧视相机参数定义

## D-007 MCT/IQC 站质检验证算法设计【已定案】

**背景**
MCT 产线标定完成后需要对标定结果做多维度质量验证，确保出厂标定参数合格。验证项包括：双目几何关系平面度、行差（极线对齐误差）、重投影误差、相邻标定板间距离一致性。

**四项验证算法**


| 验证项         | 算法方法                                                                   | 合格标准                    |
| ----------- | ---------------------------------------------------------------------- | ----------------------- |
| **平面度检测**   | 棋盘格角点检测（9×9）→ 三角化得 3D 点 → SVD 拟合平面 → 计算各点到平面距离均值和最大值                   | 距离均值/最大值低于阈值            |
| **行差检测**    | 左右目先各自去畸变 → StereoRectify → 提取棋盘格角点 → 计算左右对应点 y 方向差值均值（hangcha_sum/81） | 行差均值 < 1.22 像素（见 D-005） |
| **重投影误差检测** | 设定 3D 棋盘格点坐标 → PnP 求解位姿 → 重投影 → 计算左右目各自平均重投影误差（ave_pt）                 | 左右目误差均低于阈值              |
| **平面间距离检测** | 多个标定板位姿分别拟合平面 → 计算相邻平面中心点到平面距离 → 超阈值则标记 isok=9（不合格）                    | 距离差低于 diff_err 参数       |


**注意事项**
行差检测特殊之处：使用 StereoRectify 后的图片提角点，与其他验证项用去畸变图的处理逻辑不同（见代码注释）。

**来源** | MCT&IQC Q&A/001_滑轨检测算法逻辑/滑轨检测算法逻辑.md

## D-009 售后标定轨迹选型决策【已定案】

**背景**
售后整机标定需要在用户家庭场景（小空间）内完成，测试了多种轨迹方案。

**验证场景**

- 1.5m×1.5m 墙角
- 两面墙贴标定板（5列3行）+ 地面二维码

**轨迹方案对比**


| 方案  | 成功率  | 缺点         |
| --- | ---- | ---------- |
| S形  | 100% | 遥控困难，缺少关键帧 |
| 伞形  | 80%  | 路线冗余       |
| 工形  | 60%  | 路线可优化      |


**最终定案轨迹**

*Monet 轨迹*：逆时针旋转180° → 顺时针135° → 对角线走1.2m → 顺时针90° → 逆时针180° → 顺时针180° → 静止开始计算

**精度数据**

- 固定轮间距/半径后 yaw 误差 <0.2°
- 后轮间距最大误差 4.5cm（小场地局限）

## D-011 侧目相机外参不需标定，使用结构值【已定案】

**背景**
评估侧目相机外参是否需要通过算法标定，还是直接使用机械结构设计值。

**结论：使用结构值，无需标定**

**依据**

1. 侧目主要用于活体检测（转向盲区），双目看不到的直行不会压到
2. 危险边界依赖前视双目，侧目仅辅助
3. 侧目成像质量差，单目测距精度低，只能辅助识别，不能直接测距沿线
4. 侧目垂直 FOV=118°，安装高度18.01cm，测距盲区10.8cm

**延伸结论**：即使侧目用于建图，障碍物地图更新可依赖前视双目；侧目障碍物地图价值有限，不值得为此引入标定

**合入时间**：2026-02-05（合入 HF1.0 分支，toc.z 固定为结构值，见 D-011 / `018` bug 列表）

**来源** | `006_参数标定/017_侧目相机外参标定的意义`

## D-013 外参文件保护 + gdc_mapper 问题清单【已定案】

**背景**
割草机平台发现 gdc_mapper 存在多个工程问题，影响标定结果可靠性。

**gdc_mapper 已知 6 个问题**


| #   | 问题                                     | 影响                          |
| --- | -------------------------------------- | --------------------------- |
| 1   | bin 文件存 `/mnt/reserve/gdc/`，无法随模组携带    | 换机/换模组后失效                   |

## D-014 重投影误差检测算法 v2【已定案】

**背景**
v1 算法在世界坐标系下计算重投影误差，受 camera pose 影响误差波动大，需优化。

**算法对比**


| 版本         | 坐标系   | 误差范围          | 说明                         |
| ---------- | ----- | ------------- | -------------------------- |
| v1         | 世界坐标系 | 0.3–7.5px     | 受 camera pose 影响大          |
| **v2（选定）** | 相机坐标系 | **0.3–3.9px** | 首帧相机坐标系下3D点投影到当前帧，大部分 ≤1px |


**v2 公式**
首帧相机坐标系下的3D点 → 投影到当前帧 → 计算像素误差

**打滑检测阈值**（依据 v2）

- 正常数据集：平均误差 ≤1.65px
- 打滑数据集：平均误差 6.18px
- **阈值**：平均重投影误差 >2px 或标准差 >2px → 提示轮子可能打滑/卡死

## D-015 温度补偿对焦距影响量化【参考】

**背景**
温度变化会导致镜头焦距轻微漂移，评估是否需要引入温度补偿模块。

**量化结论**

公式：`fx_new = fx_fac + kx × deltaT`


| 场景              | deltaT | fx/fy 变化量    |
| --------------- | ------ | ------------ |
| 工厂20°C → 工作70°C | +30°   | +2.25px（最大）  |
| 工厂30°C → 工作0°C  | -30°   | -2.175px（最大） |


**参考系数**：扫地机 kx = 0.07~0.0725（随温度区间变化）

**用途**：作为是否需要引入温度补偿模块的量化判断依据（极限工况约 ±2px，视应用精度要求决定）

## D-016 相机分辨率异常根因与修复方案【已定案】

**背景**
售后标定后发现内参数据异常（W1088, H1280 vs MCT 期望 W640, H544），原因调查触发对内外参更新流程的根因分析。

**根因**
售后标定只更新外参，内参仍沿用原厂数据；驱动（宗志勇）在写入 e2 时，对图像做了长宽交换 + 分辨率除 2 的处理，导致从 e2 读出的内参与标定时使用的内参不一致。

**修复方案**
理解驱动写入规则后，售后标定流程需考虑驱动对内参的变换，确保写入 e2 和读出时的坐标系一致；具体方案与驱动侧对齐。

**来源** | `inbox/0412新增/VSLAM 进度同步 -2026`（2026-04-06 周报）；飞书：[割草机相机标定流程梳理](https://roborock.feishu.cn/wiki/LjGuwdGxCiLegzkyjEWcrtt1nUh)

## D-017 降本治具 yaw 一致性量化结论【已定案】

**背景**
为降低售后标定治具成本，测试了多种简化版标定板方案，并量化 yaw 一致性指标。

**测试结论**

| 治具方案 | yaw 极差 | 结论 |
|---------|---------|------|
| 两行 | 1.5° | 不可接受 |
| 两行 + ground | <1° | 可接受下限 |
| 三行（标准）| — | 基准 |
| 窄距三行 | 1.3° | 不可接受（宽松场景） |

**选定方案**
两行 + ground marker 方案，yaw 极差 <1° 为可接受方案；窄距三行因一致性下降（1.3°）不推荐。

**来源** | `inbox/0412新增/VSLAM 进度同步 -2026`（2026-03-30 周报）；飞书：[售后标定二维码测试](https://roborock.feishu.cn/wiki/Sdalwi7QTiX1czk3orscAMNjnHg)
