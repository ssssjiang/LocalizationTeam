# 1. 会议议程

1. 外参文件保存EEPROM，中间层以消息的形式发出

2. 标定参数的校验

3. gdc\_mapper提出一个函数，用来在线生成remap表

# 2. gdc mapper作用（当前）

1. 从EEPROM里面读取相机内参，计算R1R2P1P2Q（时间话费ms级别），算左图右图remap表（大小640\*544\*2的double），将remap表写为txt文件（这步比较慢），txt文件转bin，保存在/mnt/reserve/gdc/目录下

2. 运行时，gdc模块将bin文件读取到内存，进行每帧图片的去畸变处理

# 3. Gdc mapper当前的问题

1. 生成的bin文件保存在/mnt/reserve/gdc/目录下，文件本身本身跟双目模组绑定，无法做到文件跟随模组。

2. 目前双目模组EEPROM留4KB给石头自用，不够存放gdc\_mapper生成的bin文件（13KB）

3. 存在/mnt/reserve/gdc/目录下的bin文件有损坏可能

4. 这个bin文件是由双目模组内参生成的，本质上来说是一份数据存了两次

5. gdc\_mapper目前会替换dual\_calibration.json，造成了Monet MK2标定结果全部错误

6. 当前生成bin文件的耗时，主要是写文件（remap表写txt，txt再转bin），整个过程大概几十秒

# 4. gdc\_mapper的优化方向

1. 提出一个函数，读取双目模组内参，计算R1R2P1P2Q（ms级别），算左图右图remap表（640\*544\*2double），不写入文件，直接在内存中，供gdc模块使用&#x20;

2. 这个函数在每次启动的时候，由驱动层（？）调用&#x20;

3. 标定模式下，也需要在启动时调用这个函数，保证存gdc校正过的图

# 5. dual\_calibration.json的改动&#x20;

1. model增加double\_sphrere条目

2. 去掉R1、R2、P1、P2、Q这几项

   * 中间层每次从EEPROM读取内外参，调用俊杰的接口，计算PQ矩阵，pub出来；

   * 中间层调用gdc接口，输入内外参和计算好的PQ矩阵，计算remap矩阵；

# 6. EEPROM里面参数的使用方案：（B2）

1. 所有相关参数转为bin写入EEPROM，开机后中间层以消息的形式pub出来&#x20;

   1. 如果读EEPROM失败，读json pub出来(待定)

   2. 如果读json失败，报错机器无法启动（暂定）

2. 每次启动校验，如果EEPROM里面的SN和json里面的SN不一致，提醒用户进行外参标定（目前已有报错）

   1. 这个错误不能阻断进入整机标定模式（状态机支持）&#x20;

3. AT命令可以改EEPROM的值，有扫地机AT命令的校验方式可以复用。

   1. 每次启动如果EEPROM读取成功，都要将EEPROM里面读出的外参，备份到json文件里面

# 7. 产线标定：（B2）

1. 标定的时候所有参数写入EEPROM，然后读EEPROM，写入dual\_calibration.json。dual\_calibration.json里面的参数只作为备份

2. dual\_calibration.json放进/mnt/reserve/，此地址设定为不可写

# 8. 产线校验：（B2）

1. 所有参数从EEPROM中读出并写入dual\_calibration.json，用来走校验流程

   1. 没有保持与工作时一致，因为校验是在产线上位机进行的，没法走pub的方案

   2. 验证dual\_calibration.json里面的相机到整机外参与标定参数一致（ 代码确认，在标定时候验证）

2. 极限校正（俊杰）可以验证投影参数、畸变参数、双目之间外参的整体一致性。&#x20;

   1. 走八字的过程就需要gdc的图

3. Cam-IMU外参，IMU数据准确性在产线抽检的机械臂工站校验&#x20;

4. CAM-odom外参，校验矩阵维度、一些关键的和结构相关的值不超阈值，所有常量正常。&#x20;

   （产线标定时，上位机执行，对标定结果进行检测，所有检测项通过后才会标定成功，否则标定失败不生成标定文件dualcamera\_calibration.json）

   1. 检测左目外参变换Tocl的欧拉角和平移量与结构参数对比，需满足在容忍误差范围内；

   2. 检测右目外参变换Tocr的欧拉角和平移量与结构参数对比，需满足在容忍误差范围内；

   3. 检测相机上imu到odo外参变换Toi的欧拉角和平移量与结构参数对比，需满足在容忍误差范围内；

   4. 检测相机到odo时间延迟Time\_offset\_oc在合理的阈值范围内；

   5. 检测极限校正参数R1、R2、P1、P2、Q已经更新，不是零矩阵；

   6. 在整机功能检测站（MFT）拍摄棋盘格（或二维码）图片，依据工装结构参数估计一个相机外参，与MCT整机标定结果对比，检验最终标定结果。
