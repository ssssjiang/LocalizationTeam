## 关注场景


| 场景      | 形态                         |
| ------- | -------------------------- |
| 窄通道     | ≤ 1.5m 直通道，长 ≥ 5m          |
| L 型通道   | 90° 转弯，前后段各 ≥ 3m           |
| 坡       | 5°-35°，欧洲 30° 普遍；优先复合青苔/湿草 |
| RTK 阴影区 | 屋檐 / 树荫 / 高墙 / 基站本身在屋檐下    |
| 桩位变化    | 三种时态：建图中 / 割草中 / 桩上        |


**复合场地优先**（一户覆盖多场景）：

- L 型 + 坡度
- 大坡 + 青苔
- **基站本身在屋檐下**（A-7 黄金场地）

## 现象记录

> 现象记录档位为本次约定的统一标准（源方案中"位置精度"为定性描述，本次细化为四档便于现场快速判断和事后汇总）。

| 档   | 含义                |
| --- | ----------------- |
| 准确  | APP 显示位置与实际基本一致   |
| 有偏差 | 看得出偏，但还可接受        |
| 明显偏 | 一眼看出错位            |
| 离谱  | 严重不符（如显示在通道里实际撞墙） |

记录方式：APP 截图 + 第三方相机拍实际位置 + 文字描述

# Track A：竞品测试用例

## A-1 窄通道完整流程

> 来源：
> - 5 步流程 + 观察 1/2/3/4/6：源方案 §6.1 + §6.2 完整对应
> - 观察 5（中段拖拽打滑）：本次扩展，借鉴源方案 §4.1-2.d「建图过程中拖住机器使轮子打滑」，把它叠加到通道场景下

**场地**：窄通道宽 ≤ 1.5m、长 ≥ 5m，两端各有 ≥ 5×5m 草坪。**A 离桩很近，B 离桩很远，A↔B 跨两道窄墙通道**（源方案 §6.1 要求）

**运动序列**（一次跑完，3 轮）：

```
P1  出桩 → A 区遥控建图（A 离桩很近）
    观察1：能否建图？是否引导双向建通道（仅 RTK）？

P2  从 A 遥控通过通道到 B 区，建 B 区（B 离桩很远，跨两道窄墙通道）
    通道中段：抱起 → 旋转 90° → 原地放回
    观察2：是否显式重定位（语音/APP 提示）？行为（走方块/原地转圈）？耗时？后续能否走出通道？
    继续遥控通过剩余通道
    观察3：通道内 APP vs 实际感受？进通道/通道内是否磕碰？磕碰时 APP 是否仍在通道？
    在 B 区建图

P3  建立通道 B → A，走出窄通道后立即结束
    观察4：能否成功建通道？

P4  桩出去 B 区选区割草，机器自动反向通过通道
    通道中段（扩展项）：拉住机器尾部 5 秒 → 释放
    观察5：打滑后是否有定位找回动作？能否继续？APP vs 实际？
    到 B 区割草

P5  割草完成 → 自动回桩，再次反向通过通道
    中段：抱起 → 旋转 90° → 放回
    观察6：同观察2（反向通过通道时被劫持的行为，源方案 §6.2-8）
```

**机型**：RTK 款 + 激光款都做。源方案明确"激光雷达竞品无需测"的项：观察1 中的"是否引导双向建通道"、观察3 中的"通道内 APP 显示 vs 实际位置"、"割草时能否进入窄通道"。

## A-2 L 型通道完整流程

> 来源：本次新增。
> - L 型通道场景：海外踩点分享（2026-04-15）多次提到 L 型通道在欧洲住宅区常见；源方案 §6 仅描述直线窄通道，无 L 型相关章节
> - 观察 1/2/3：参照 A-1 窄通道的"通道行为 + 中段劫持 + 反向通过"思路，针对 L 型转弯场景设计
> - "带坡复合"：海外踩点分享提到 L 型通道常带坡度

**场地**：L 型通道，转弯 90° ± 10°，前后段各 ≥ 3m。**优先：带 ≥ 15° 坡度**（如能找到带坡场地，可顺便观察坡上行为；不带坡场地跳过坡相关观察）

**运动序列**（一次跑完，3 轮）：

```
P1  出桩 → 建 A 区

P2  从 A 遥控通过 L 型通道到 B 区
    通过转弯时
    观察1：转弯处轨迹是否连续？前后位置/角度感受？是否磕碰角壁？
          （仅带坡场地）是否打滑？是否触发重定位？
    转弯完成后停下
    在转弯位置：抱起 → 旋转 180° → 放回
    观察2：是否显式重定位？行为？180° 旋转后朝向是否识别正确？能否走出通道？
    继续到 B 区建图

P3  建立通道 B → A

P4  B 区选区割草 → 反向通过 L 型回桩
    观察3：反向到达转弯点 APP 朝向 vs 实际？是否磕碰？是否触发重定位？反向通过成功率？
```

**机型**：通用

## A-3 坡度完整观察

> 来源：
> - 观察 1/2「沿/垂直坡梯度方向割草，关注弓字轨迹是否平直」：源方案 §2「斜坡（RTK 阴影区域）」+「斜坡（空旷更优）」两类共同要求
> - 观察 1/2「是否漏扫」（仅激光）：源方案 §2「斜坡（空旷更优）」激光列明确要求
> - 观察 3：本次新增，复合场景。复合背景来自海外踩点分享（30° 大坡 + 青苔湿草普遍）；观察指标"是否丢定位 / 是否出界"借鉴源方案 §2「贴墙、贴障碍物、贴灌木打滑」激光列要求

**场地**：≥ 15° 坡，长 ≥ 5m。优先包含 ≥ 25° 坡 + 湿草（如海外踩点常见的青苔湿地）

**运动序列**（每个坡 1 次）：

```
P1  在坡面上正常建图

P2  弓字方向 = 沿坡梯度（上下） → 割草 ≥ 5 弓字
    观察1：弓字轨迹是否平直？是否打滑（次数）？APP 轨迹 vs 实际？是否漏扫（仅激光）

P3  弓字方向 = 垂直坡梯度（横切） → 割草 ≥ 5 弓字
    观察2：同观察1（垂直梯度方向）

P4  （扩展项，仅当有大坡 ≥ 25° + 湿草场地时）正常割草
    观察3：是否丢定位？是否出界？是否触发刀盘停转/安全保护？APP vs 实际感受
```

**机型**：通用。观察 1/2「是否漏扫」仅激光记录（源方案要求）

## A-4 打滑完整观察

> 来源：
> - 观察 1：源方案 §4.1-2.d 三个子问题之一「遮挡（视觉定位传感器）加打滑后精度怎么样」。源方案前两个子问题（纯打滑后是否有重定位、纯打滑后建图定位精度）已被 K-2 实测覆盖（Navimow X3 RTK 盖锅子+空转），本次仅做"打滑+视觉遮挡"组合，避免重复。源方案明确要求「**RTK 机器需要在 RTK 阴影区域进行这项测试**」
> - 观察 2：源方案 §2 场景表「贴墙、贴障碍物、贴灌木打滑」（仅激光列），观察指标「是否丢定位 / 是否出界」

**场地**：**RTK 阴影区**（信号好会自动校正掩盖打滑表现，源方案明确要求）。优先：阴影 + 贴墙/灌木复合场地

> 已知（K-2，Navimow X3）：RTK 盖锅子+抬后轮空转时，定位会"隔一段时间刷新回来"（疑似机身 GPS 兜底）；不盖锅子时打滑也一直往前走。源方案 §4.1-2.d 前两个子问题已被覆盖；本次只补**视觉遮挡组合**这个未做的第三子问题。

**运动序列**：

```
P1  在 RTK 阴影区开始建图

P2  建图中：用黑胶带遮挡双目（视觉定位传感器） → 拉住机器尾部 5 秒（使轮子打滑） → 释放
    观察1：打滑后是否有定位找回（重定位）动作？行为（转圈/特定轨迹/仅停顿）？耗时？
          视觉失能下定位精度如何（APP vs 实际）？是否有提示（语音/文字）？
    揭开遮挡，继续建图

P3  （仅激光机型）贴近墙体/障碍物/灌木开始割草，自然产生打滑
    观察2：是否丢定位？是否出界？（观察 ≥ 5 分钟）
```

**机型**：
- 观察 1 RTK + 激光都做（**RTK 机型必须在 RTK 阴影区**，源方案要求）
- 观察 2 仅激光机型（源方案 §2 仅在激光列要求）

## A-5 桩位变化完整观察

> 来源：
> - 观察 1/2/3：源方案 §2「建图过程中桩位置移动」「割草过程中桩位置移动」「桩上桩位置移动」
> - 观察 4/5/6：源方案 §4.3-3/4/5（换桩 APP 不更新/同步 + 假成功自愈）
> - 观察 7：源方案 §4.3-6（搬桩到无共视 B 区，仅激光）

**场地**：草坪 ≥ 5×5m。观察 4/5/6 需 RTK 阴影区（仅 nRTK 机型测，源方案要求）；观察 7 需 A 区 + 与 A 无共视的 B 区

**运动序列**（拆 4 段）：

### 第 1 段：建图过程中桩位置移动

```
出桩开始建图 → 进度约 50% 时物理移动基站（位置变化明显即可）
观察1：能否继续建图？能否回桩？
继续建图 → 尝试回桩
```

### 第 2 段：割草过程中 + 桩上桩位置移动

```
完成建图 → 启动割草
割草中：物理移动基站
观察2：能否继续割草？是否出界？是否正常回桩？
等机器自动回桩，停在桩上
桩上：物理移动基站
观察3：同观察2（桩上时态）
后续启动割草观察
```

### 第 3 段：直接更换桩的位置（nRTK 机型 + RTK 阴影区）

> 与第 1/2 段的"桩小幅扰动"不同，本段是"桩搬到新位置"，模拟用户重新安装桩

```
完成建图（在阴影区）
直接更换桩的位置（搬到一个新位置），APP 上不做任何操作
把机器抱/开到新桩附近
触发重定位
观察4：重定位是否成功？定位的位置是否准确？是否假成功？

在 APP 上操作一下"更换桩位置"，使得 APP 上显示的桩位置更新到新位置
机器再回到新桩附近，再次触发重定位
观察5：同观察4（APP 同步桩位后）

如 观察 4 或 观察 5 重定位假成功：
不重启，继续长时间割草，直到回桩
观察6：定位是否可以恢复？回桩点位置感受？
```

### 第 4 段：搬桩到无共视的 B 区域（仅激光机型，2 次）

```
A 区域建图成功 → 搬桩到没有共视关系的 B 区域
机器在 B 区域开始割草 → 一段时间后触发重定位
观察7：是否会成功还是一直尝试重定位？是否会重定位到新 B 图上？是否建立新地图？
```

**机型**：
- 观察 1/2/3 通用
- 观察 4/5/6 仅 nRTK 机型 + 必须 RTK 阴影区（源方案要求）
- 观察 7 仅激光雷达机型（源方案要求）

## A-6 抱起机器后的重定位

> 来源：
> - 观察 1：源方案 §4.3-1（抱起机器后重新放置在草坪上）
> - 观察 2a/2b：源方案 §4.3-2（放置在区域 1 之外，区域 1 建图已完成 / 未完成两子工况）
>

**场地**：完整草坪 + 区域外可放置位置

> 已知（26.3 §2.4 九号 i206 1+1 RTK）：RTK 良好下抱起放回 1m / 转 3 圈 / 朝向偏 90° 三种工况，轨迹和朝向都能实时更新（详见末尾已做项 §K-3）。本次重点：**激光机型**补测抱起放回行为 + 所有机型测搬到地图外。

```
机器在草坪正常工作
抱起机器 → 重新放回草坪上
观察1：机器是否会有重定位动作？只需要转圈，还是会走一些特定轨迹？耗时？
```

```
工况 b：区域 1 建图未完成

```

**机型**：

## A-7 RTK 不好 + 搬动场景下的重定位

> 来源：飞书讨论 2026-04（宋姝、杨倩、李哲）。源方案无对应章节，是本次新增。**接力杨倩前期工作**——出发前必联系杨倩同步进度。

**目标**：测试 RTK 不好（信号差 / 被遮挡）+ 机器被搬动时，能不能重定位成功；以及是不是靠视觉

**场地**：基站本身在屋檐下的房子（RTK 信号天然差）

> 已知（26.3 §2.4 九号 i206）：RTK 阴影 + 抱起搬运 1m（朝向一致）→ 定位**不更新**，闭合时地图就不正确，视觉优化基本没变化，跑一圈整体偏移 1m 不可用（详见末尾已做项 §K-4）。本次重点：**关机重启**、**地图外**、**双目遮挡**这三个未做维度。

**两个时态分别测**（区别：建图时无旧地图，割草时有旧地图）：

### 建图时（无旧地图）

**前置**：本次开机后从未建过图


| 用例   | 操作                                                                                  | 想回答                       |
| ---- | ----------------------------------------------------------------------------------- | ------------------------- |
| 7M-1 | 建图前盖基站 → 手动关机重启 → 关机后搬运至屋檐下 → 开机 → 等冷启动结束（机身灯由置灰转黄）→ 重新触发建图（注：源方案已知关机后无法续建，需重新触发） | RTK 差 + 搬动后，重新触发建图能否成功    |
| 7M-2 | 同 7M-1，但搬到**地图外区域**（不在已建地图覆盖范围）                                                     | 完全陌生位置能否开始建图              |
| 7M-3 | 同 7M-1，开始前用黑胶带遮挡双目                                                                  | 失去双目视觉后能否建图（验证是否依赖双目视觉） |


### 割草时（有旧地图）

**前置**：已存在一张完整地图


| 用例   | 操作                                            | 想回答                      |
| ---- | --------------------------------------------- | ------------------------ |
| 7W-1 | 割草前盖基站 → 手动关机 → 关机后搬运至屋檐下 → 开机 → 等冷启动结束 → 启动割草任务 | 有旧地图时，RTK 差 + 搬动后能否重定位回旧地图 |
| 7W-2 | 同 7W-1，但搬到**地图外区域**                             | 离开旧地图覆盖范围能否成功               |
| 7W-3 | 同 7W-1，开始前用黑胶带遮挡双目                              | 失去双目视觉后能否重定位（验证是否依赖双目视觉）  |


每用例 3 次。

**观察记录**：


| 项      | 记什么                                         |
| ------ | ------------------------------------------- |
| 是否成功   | 是 / 否                                       |
| 耗时     | 粗略秒数                                        |
| RTK 状态 | 全程记录：4 / float / single / none              |
| 位置感受   | 遥控到已建图的边上，看 APP 与实际是否一致：准确 / 有偏差 / 明显偏 / 离谱 |
| 是否假成功  | 位置明显偏即视为假成功                                 |


**机型**：优先九号；其他竞品 + 我方机器同步做对比

**判据**：

- 7M / 7W 同条件下表现差异 → 旧地图是否帮助重定位
- 7x-1 成功 + 7x-3 失败 → **双目视觉是关键依赖**
- 7x-1 成功 + 7x-3 也成功 → 不靠双目视觉（可能靠 RTK 弱信号 / 轮速 / IMU / 激光等其他传感器）
- 7x-2 成功 → 可在完全陌生位置初始化（不限于已知地图覆盖范围）

## A-Q 定量漏割（可选）

> 来源：源方案 §5.1 场地需求 + §5.3.1 调头漏割 + §5.3.2 行间漏割。源方案此处定义了完整的指标体系（详见示例表表头），但因示例表无任何实测数据回填，本次属首次实测。
> 仅在能找到 10×10m 规整草坪时做，不强求。

**场地**（源方案 §5.1）：在草地遥控建一个外框 10m × 10m 的图，使弓字的长度为 10m。

**测试项与指标**（源方案 §5.3 示例表表头）：

# 附录：已有结论（出发前阅读，可跳过对应重复测试）

> 出发前阅读，现场遇到对应场景时**可跳过对相同机型的重复验证**，把时间留给其他厂商和未做的维度。
> 来源缩写：
>
> - **源**：源测试方案 `割草机竞品测试-建图定位.md`（PDF 红字结论）
> - **26.1**：26.1-德国竞品测试
> - **26.3**：26.3-德国竞品测试

## K-1 建图行为与 EFLS（来源：源 §4.1，机型：九号 i 系列）

- 割草顺序：**沿边 2~3 次 → 沿禁区 → 弓字**（不同机型沿边次数有差异）
- 建图后**没有自主沿边**
- **没有地图中间区域的探索**
- **EFLS 视觉地图优化**：用户可选开启；**仅 RTK 款有**；信号好不触发；信号差才触发；优化时间几分钟；**优化前后地图基本无变化**（搬运情况下尤其明显）

> 影响：A-7 不必再期待视觉优化能救回 RTK 阴影下的偏移；其他用例的"建图全过程"前置预期可参考。

## K-2 RTK 盖锅子 + 抬后轮空转（来源：26.1，机型：九号 Navimow X3 RTK）

- 锅子盖住 1min 不会报错，能一直建图
- 抬后轮空转：会一直往前跑，过一阵子定位点又**刷新回来**重新建图（**RTK 顶着锅子时**）
- 拿掉锅子后空转：**不会再刷新回来**，会一直往前走（**RTK 不顶锅子时**）
- 重新盖上锅子继续空转：又会**隔一段时间定位刷新一次**

**完整建图全程盖锅子**：信号差的地方偏移很大；信号好的地方隔一段时间会**跳变校准一次**，跳变后位置相对准（疑似机身 GPS 兜底定位）

**割草中盖锅子**：地图飘 4-5 个机身后开始进入脱困；放开机器继续运行，会**一直用飘了的地图割草**；**拿走锅子后定位会闪回正确位置**

> 影响：A-4 观察1（拖拽打滑）已有充分实测；本次 A-4 重点放在**叠加双目遮挡**和**激光机型**。

## K-3 RTK 良好下抱起搬运 / 转圈（来源：26.3 §2.4，机型：九号 i206 1+1 RTK）


| 操作                       | 结果                                                               |
| ------------------------ | ---------------------------------------------------------------- |
| RTK 良好 + 抱起搬 1m + 朝向一致   | 搬运时轨迹**实时更新**，朝向与实际一致；生成轨迹验证一致；**地图可用**                          |
| RTK 良好 + 转 3 圈 + 朝向一致    | 搬运转圈时轨迹实时更新；生成地图时**优化掉了搬运后这段**（地图优化问题非设定逻辑）；矩形轨迹一致               |
| RTK 良好 + 转 3 圈 + 朝向偏 90° | 转圈时轨迹实时更新；搬运后机器朝向与轨迹**稍有偏差**（大方向一致）；生成地图时搬运轨迹**优化偏差较大**；矩形内轨迹仍一致 |


> 影响：A-6 观察1（抱起放回）的 RTK 良好场景已实测，本次重点放在**激光机型**补测和地图外。

## K-4 RTK 阴影下抱起搬运（来源：26.3 §2.4，机型：九号 i206）


| 操作                                               | 结果                                                      |
| ------------------------------------------------ | ------------------------------------------------------- |
| RTK 阴影 + 抱起搬 1m + 朝向一致（自然阴影）                     | 搬运后定位**未更新**；闭合时地图就不正确；视觉优化基本没变化；跑一圈地图整体偏移 1m，**实测不可用** |
| RTK 阴影 + 抱起搬 1m + 朝向偏 90°（人为盖信号灯制造阴影：搬运前盖、搬运后拿走） | 定位**有同步**                                               |


> 注：两条结果不同，疑似与"信号差是自然产生还是人为盖灯"有关——人为盖灯解除后立即恢复 RTK，定位能跟上；自然阴影持续存在，定位无法补救。
> 影响：A-7 不必再做"RTK 阴影 + 搬运 1m + 不关机"的基础验证；本次重点放在**关机重启 + 地图外 + 双目遮挡**三个未做维度。

## K-5 Luba 续建（来源：26.3 §2.4，机型：松灵 Luba nRTK）

- nRTK 信号好下，建图过程中搬运、拖拽、抬起 1m，都**不会失败**，朝向轨迹**实时更新**且正确
- **关键限制**：Luba 是"检测到闭合则自动完成"的逻辑；搬运中如果检测到闭环出现，**报失败**（定位丢失）
- 因此 **Luba 不支持 8 字型地图**

> 影响：A-1 / A-6 在 Luba 上要避免触发误闭合的运动序列。

## K-6 各机型简单重定位 / 续建支持情况（来源：26.1 各机型小节）


| 机型                     | 续建支持                         | 备注                                              |

| 九号 Navimow X3          | 支持                           | 拉出 3m 内回界继续割草；拉出 4m 后**先重定位再报错**；RTK 盖锅子详见 §K-2 |
| 九号 Navimow i108        | 26.1 表格留空                    | 未实测                                             |
| Sunseeker X7           | 26.1 表格留空                    | 未实测                                             |
| 松灵 Luba mini AWD LiDAR | 关闭后台重新打开可续建；**建图时搬动机器，建图失败** | 同 §K-5 闭合误检                                     |
| YUKA                   | 26.1 表格留空                    | 未实测                                             |


> 影响：A-1 / A-2 / A-6 在 Luba 上需要注意"搬动直接失败"的边界；i108 / X7 / YUKA 这些机型本次需要补测。

## K-7 充电桩位移后回充（来源：26.1 Luba mini）

- 只要红外能对接到就可以回充，但**地图充电桩位置不更新**
- 第一次失败，第二、三次成功

> 影响：A-5 桩位变化的"是否能回桩"在 Luba 上已知部分结论；A-5 需测重定位行为而非回充行为。
