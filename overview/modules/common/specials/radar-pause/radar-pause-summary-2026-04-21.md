## 一、核心方案：断流 2.0

**目标**：导航无感，断流后能恢复则保持无感运行；无法恢复则触发重定位，保障机器作业连续性。

**对应决策**：[D-008 雷达断流处理 2.0](../../../../../teams/laser/modules/mslam/decisions.md)
**前序方案**：D-007（断流 1.0，已被 D-008 替代）

---

## 二、分场景处理策略

| 场景    | 断流时       | 恢复后                                       |
| ----- | --------- | ----------------------------------------- |
| **建图** | 机器**停机**  | 原地停机做局部重定位，失败则发起**主动全局重定位**               |
| **割草** | 机器**降速**  | 动态局部重定位，成功则提速恢复，失败则发起全局重定位               |

> 注：建图模式 1s 即停机；定位模式 1s 降速、6s 才停机。详见 D-008 表格。

## 三、关键执行细节

| 项目           | 详情                                                        |
| ------------ | --------------------------------------------------------- |
| **断流判定阈值** | 断流时长超过 **500ms** 即判定为断流（与 D-007 历史一致；D-008 起降速触发改为 1s） |
| **融合模块**     | 断流期间正常发布位姿                                                |
| **多线 SLAM**  | 正常进行 odom 递推，但**不修正、不发布位姿**                              |
| **恢复时**      | 使用自身递推的 pose 作为局部重定位初始位姿，**不再使用导航的 setpose**            |
| **局部重定位**    | 支持**动态局部重定位**，不强求机器停下                                    |
| **超时保护**     | 断流期间计时，超过 **6s** 直接停机报错                                  |
| **重定位中断（套娃）** | 局部重定位计算时若再次断流，直接返回失败，计时累加，直接升级全局重定位                     |

> 对应决策细节：D-008（兼容设计 ps 章节） + D-015（轮速引入断流补偿）。

## 四、雷达断流原因分析

| 原因           | 说明                                                                |
| ------------ | ----------------------------------------------------------------- |
| **GPS 报文阻塞** | 模拟 GPS 报文因 AP 实时性出现阻塞，最大可能有几帧阻塞                                  |
| **时间戳回退**    | 雷达收到延迟的 GPS 报文，导致时间戳回退引发断流                                        |
| **硬同步失效**    | 时间同步问题，模拟信号无法满足硬同步要求                                              |
| **厂家强制关闭**   | **大疆厂家会强制关闭 2s** 也会造成断流（来自本次工作汇总，非现有 D-008 文档记录）                 |
| **SDK 加印日志** | SDK RTM 分支加打印日志影响实时性，是 2026-01 底断流频率显著上升的诱因之一（见 D-008 历史背景章节）  |

> ⚠️ 「大疆厂家强制关闭 2s」此前未在 `decisions.md D-008` 的根因章节出现，本次专题汇总首次记录；建议向飞书群讨论 / SDK 文档复核确认。

## 六、重定位算法优化（断流恢复路径上的关键升级）

| 优化项               | 效果                                                | 仓库映射                       |
| ----------------- | ------------------------------------------------- | -------------------------- |
| **MULLS 框架替换**    | 树木特征点均匀提取，杂点少；重定位耗时从 **6s 降到 200ms**            | **D-034**（新建）            |
| **双地图模式**         | 关键帧地图用于特征匹配 + 体素地图用于 ICP 匹配；整体约 **10s**         | **D-034**（新建）            |
| **自动退出策略**        | 耗时达到阈值自动退出，防止算法卡死                                  | **D-034**（新建）            |
| BBS 搜索范围扩展（±15m / 360°） | 搬动 / 回桩等场景成功率提升                                | D-017                       |
| 双局部地图（窄通道）       | 定位用全点云图 + 重定位用 2.5m 内仅地面点                          | D-028                       |
| BBS3D 引入主流程       | 树叶杂乱场景 corner case 兜底（2026-04-17 bug #496132）   | timeline / W16 周报          |
| 重定位建图过程降采样      | 控耗时（与 BBS3D 引入同期合入）                                | timeline / W16 周报          |

## 七、后续优化方向

| 优化项         | 内容                                                                                |
| ----------- | --------------------------------------------------------------------------------- |
| **减少重定位次数** | 用户搬起放下割草机时，在满足条件下减少重定位（→ D-022 / D-025）                                       |
| **暂停时优化**   | 暂停时有 lidar 且连续则计算，不给融合发 pose；断流则恢复原有 pose 再发起主动重定位（→ D-016）                  |
| **脏污检测规则**  | 连续 **10 帧点数 < 300** 则报脏污；需区分遮挡与雷达故障（与 D-020 速腾 SDK count/intensity 规则联动；本次新增） |
| **斜坡补偿优化** | 高度积分 + 重力对齐角度积分方向（→ P-007，断流 2.0 已知风险）                                          |
| **奇异场景**    | 长时间断流 / 频繁断流 / 套娃断流的端到端验证（→ P-006）                                              |
