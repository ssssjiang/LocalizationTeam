## 2. 测试项清单（细粒度）

### F-T-001 RTK 信号差行为 — 局部建图盖锅子

- **场景 / 专题**：RTK 信号丢失（人造，盖锅子模拟）
- **涉及竞品**：
  - 九号 X3：锅子盖住 1min 不报错、能持续建图；4 分多钟抬后轮空转，机器一直往前跑、隔一段时间定位刷新回来；6:15 拿掉锅子后空转，定位不再刷新；7 分钟盖回锅子又开始隔段时间刷新 [26.1 §Navimow X3 - 局部建图 RTK 盖锅子]、[海外分享]
- **关键指标**：报错延迟（**>1min 不报错**）；信号丢失时定位刷新周期（**未量化**）
- **现象解释**：盖锅子时疑似切到机身 GPS 单点定位做周期性纠偏；不盖锅子但轮子空转时不会刷新（推测：信号好时主要靠 RTK + 里程计，不主动用 GPS 校准）

### F-T-002 RTK 信号差行为 — 全程盖锅子建图比对老图

- **场景 / 专题**：RTK 持续信号差下的视觉/GPS 兜底
- **涉及竞品**：
  - 九号 X3：全程盖锅子原图重建可实时比对轨迹偏移；信号差地方偏移大；信号好处隔段时间跳变校准、跳变后位置相对准（疑似机身 GPS） [26.1 §Navimow X3 - 完整建图时 RTK 盖锅子]、[海外分享]
- **关键指标**：跳变前后位置差异（**未量化**）；最终建图相对老图的整体偏差（**定性：偏 1-2m，地图整体不可用于割草**）
- **现象**：建完图后割草飘很多，"实际不能用"

### F-T-003 RTK 信号差行为 — 建图盖锅子 / 割草不盖锅子

- **场景 / 专题**：建图与执行阶段的信号一致性
- **涉及竞品**：
  - 九号 X3：开着 GPS 去割扣着锅子建的地图，实际地图偏差挺大、不能用 [26.1 §Navimow X3]
- **结论**：信号差时建出的图，即使后续信号恢复也无法补救，地图整体不可用

### F-T-004 RTK 信号差行为 — 割草中盖锅子

- **场景 / 专题**：执行阶段 RTK 临时丢失
- **涉及竞品**：
  - 九号 X3：规划中扣锅子提起轮子空转，地图飘 4.5 个机身后进脱困；放开机器继续运行，会一直用飘了的地图割草 [26.1 §Navimow X3 - 割草中 RTK 盖锅子]
- **关键指标**：脱困触发的飘移阈值 = ~4.5 个机身

### F-T-005 RTK 信号差行为 — 拿走锅子定位闪回

- **场景 / 专题**：RTK 信号恢复
- **涉及竞品**：
  - 九号 X3：拿走锅子后定位会闪回正确定位 [26.1 §Navimow X3]
  - 九号 X3：拿走锅子信号好后，提起轮子空转定位不会飘 [26.1 §Navimow X3]
- **结论**：信号良好时不主动用 GPS 校准；信号差时启用 GPS 兜底

### F-T-006 nRTK ↔ RTK 切换（YUKA 专项）

- **场景 / 专题**：定位源切换
- **涉及竞品**：
  - 松灵 Yuka：nRTK ↔ RTK 互切**不需要重新建图**；nRTK 转 RTK 后 UI 出现 RTK 标识；首次进入界面默认非 nRTK 需手动切；信号源 4G/Wi-Fi 可选但**不动态切换**（信号差也不切） [26.1 §YUKA]、[海外分享]
- **关键指标**：切换是否需重建图（否）；切换是否带来定位中断（**未明示**）

### F-T-007 nRTK 网络中断后割草持续时间

- **场景 / 专题**：nRTK 服务断流容忍度
- **涉及竞品**：
  - 松灵 Yuka：nRTK 模式下关闭 4G & Wi-Fi 后仍可割草约 23min，然后报"RTK 基站未就绪" [26.1 §YUKA]
  - 松灵 Luba：nRTK 强绑定 4G，4G 过期则 nRTK 失效 [海外分享]
- **关键指标**：网络中断容忍 ≈ 23min（YUKA）

### F-T-008 重定位 / 续建 — RTK 良好场景

- **场景 / 专题**：重定位与续建
- **涉及竞品**：
  - 九号 i206 RTK 1+1（[26.3 §2.4 九号 1+1 款 RTK]）：
    - 搬 1m + 朝向一致 → 轨迹**实时更新**，搬运后朝向一致；生成图实测**一致**，地图可用
    - 搬 + 转 3 圈 + 朝向一致 → 实时更新；生成时优化掉搬运段；矩形轨迹实测一致
    - 搬 + 转 3 圈 + 朝向偏 90° → 实时更新；机器朝向与轨迹大方向一致但角度有偏差；生成时搬运段优化偏差较大；矩形内轨迹一致
- **关键指标**：搬运容忍距离 ≥1m；转圈容忍 ≥3 圈；朝向偏 90° 仍可续建但有角度残差
- **图片证据**：[26.3 §2.4] image-12 ~ image-21
- **关联现有体系**：与 `specials/pause-resume-reloc/` 重定位逻辑专题对照

### F-T-009 重定位 / 续建 — RTK 阴影场景

- **场景 / 专题**：重定位与续建（信号差）
- **涉及竞品**：
  - 九号 i206 RTK 1+1 ([26.3 §2.4])：
    - RTK 阴影 + 搬 1m + 朝向一致 → 搬运时定位**未更新**；闭合时地图已不正确；EFLS 视觉优化后基本无变化；实测跑一圈地图整体偏 ~1m **不可用**
    - RTK 阴影 + 搬 1m + 朝向偏 90° → 同上，定位不同步
- **关键指标**：阴影区下定位偏移 ~1m；视觉优化在阴影区**无效**
- **图片证据**：[26.3 §2.4] image-22 ~ image-30

### F-T-010 重定位 / 续建 — Luba nRTK

- **场景 / 专题**：重定位与续建（nRTK + 视觉）
- **涉及竞品**：
  - 松灵 Luba：nRTK 信号好时搬运/拖拽都不会失败、轨迹和朝向自动更新；但因为 Luba 是"检测到闭合即自动完成"逻辑，搬运中若触发闭环检测则报"定位丢失"建图失败；不支持 8 字型地图 [26.3 §2.4 Luba nRTK 款]、[方案 §4.1]
- **关键指标**：续建支持搬运 1m / 抬起拉拽；闭合误检导致建图失败

### F-T-011 出界判定（与定位边界一致性强相关）

- **场景 / 专题**：出界（已确认纳入）
- **涉及竞品**：
  - 九号 X3：拉出 3m 内回到界内继续割草；超 4m 先重定位再报错 [26.1 §Navimow X3 - 将机器拉出界]
  - 九号 i2 (激光)：慢推 3m 报错；抬后轮拉超 3m 后做重定位再正常回充 [26.3 §15 - 九号激光款 i2]
  - 九号 RTK 款（i206 / X3 同行为）：拖出会直接报；拉后轮直接返回；距离 ~2-3m；割草中搬至室内会"一直挣扎走不停"，手动暂停后再启动报错 [26.3 §15 - 九号 RTK 款]
  - 松灵 Yuka Mini：出界很严格，1.2m 即报；遥控回到 1m 内不动，需到 0.4-0.6m 才能正常启动 [26.3 §15 - Yuka Mini]
- **关键指标**：出界报错距离阈值（九号 ≈3m，Yuka Mini ≈1.2m）；恢复启动距离阈值（Yuka Mini 0.4-0.6m）
- **图片证据**：[26.3 §15] image-59、image-60、IMG_8220、出界报错信息.PNG

### F-T-012 寻桩 / 充电桩位移（与定位地图一致性强相关）

- **场景 / 专题**：寻桩（已确认纳入）
- **涉及竞品**：
  - 九号 X3：充电舌头挡住会重新上下桩 7-8 次后报错（stop 可清除报错）[26.1 §Navimow X3 - 充电]
  - 九号 i206：左右挪 ~1.5m 内能找回，2m 悬；上桩成功后退桩报"充电桩位置变化" [26.3 §11]
  - 松灵 Luba：只要红外能对接到就能回桩，但**地图充电桩位置不更新**；3 次实测：第 1 次失败、第 2/3 次成功 [26.1 §松灵 - 充电桩移位后回充]、[海外分享]
- **关键指标**：寻桩位移容忍 ~1.5-2m；松灵地图位置不随实际更新（**已知短板**）

### F-T-013 围困场景下的定位行为（已确认纳入）

- **场景 / 专题**：围困
- **涉及竞品**：
  - 松灵 Luba：被障碍物围困一圈，从始至终未报错，在内部尝试脱困、割草进度也减小；预规划路线持续显示在工作；最后会"撞一下"做二次确认（疑似通过安全检查走相对安全方向撞墙判断）[26.1 §松灵 - 围困视频]、[海外分享]
- **结论**：围困场景不报"定位异常"，靠脱困策略解；定位输出不主动失效

### F-T-014 单点 GPS 兜底（理论 / 现象关联）

- **场景 / 专题**：多源融合兜底策略
- **涉及竞品**（理论讨论 + 九号 X3 现象）：
  - 九号 X3 信号差时跳变现象推测使用全频单点 GPS（[海外分享]）
  - 单点 GPS 精度参考（口述）：曹丽娜 — 最差 4.5m / 均值 2-3m；李圳 — 全频单点 1.5m 内；李欢 — Vasa 防盗实测 5-10m
- **状态**：**未量化的假说**；本批材料无独立验证实验

### F-T-015 退桩建图的 RTK 信号阈值

- **场景 / 专题**：退桩建图触发条件（信号阈值）
- **涉及竞品**：
  - 九号：**退桩建图要求 RTK = 4（信号绝对好）才支持**；信号未达阈值则不允许退桩进入建图态 [产品反馈 2026-04-18 §1.1]
- **关键指标**：固定门控阈值 RTK 状态 = 4（FIX 解）
- **现象解释**：九号将"建图入口"作为强信号守门，避免后续地图整体不可用（与 F-T-002 / F-T-003 现象呼应：信号差时建出的图无法补救）
- **关联**：与 NR-P0-2（持续信号差兜底）形成"入口策略 vs 兜底策略"对照

### F-T-016 通道建图对通道起点的定位要求

- **场景 / 专题**：通道建图触发条件（重定位门控）
- **涉及竞品**：
  - 九号：**草坪间通道**建通道时对**通道起点定位要求高**，需重定位成功才能建通道；但即使信号不佳（铁锅盖住基站）**也能重定位成功** [产品反馈 2026-04-18 §1.2]
  - 松灵 Luba（对照）：通道场景 App 显示位置与实际偏 1-2 个机身、设置通道起点失败 [26.1 §松灵 - 建图]
- **关键指标**：通道起点重定位成功率（**未量化**）
- **现象解释**：九号在通道入口用"重定位成功 + 不依赖 RTK FIX"做门控，这是与 F-T-015 不同维度的策略（一个用 RTK 阈值、一个用重定位结果）

### F-T-017 回桩通道拓扑约束

- **场景 / 专题**：通道拓扑约束
- **涉及竞品**：
  - 九号：**只支持一条**充电桩↔草坪通道（拓扑硬约束）[产品反馈 2026-04-18 §1.1]
  - 松灵 Luba（对照）：可设置 **2 条回桩通道**（每个区域一条）[26.1 §松灵可设置 2 个回桩通道]
- **结论**：九号 vs Luba 在回桩通道数量上是 1 vs N 的策略差异

### F-T-018 RTK 差 + 搬运耦合下的回桩失效模式

- **场景 / 专题**：耦合失败模式（信号差 × 物理扰动）
- **涉及竞品**：
  - 九号：RTK 信号特别不好时大部分情况下都能正常回充；但**发生搬运等情况时偶尔会方向混乱出界** [产品反馈 2026-04-18 §1.3]
- **关键指标**：方向混乱出界发生率（**未量化，"偶尔"**）
- **现象解释**：单一信号差不致命；信号差 × 突发物理扰动叠加时朝向估计退化导致回桩路径错位

### F-T-019 退桩动作 — 三点定位（Luba 桩前信息收集）

- **场景 / 专题**：退桩定位策略
- **涉及竞品**：
  - 松灵 Luba：退桩后做**三点定位**，收集丰富的桩前信息 [产品反馈 2026-04-18 §2.1]
- **现象解释**：Luba 用退桩阶段的多点观测建立桩前局部地图先验，后续回桩用这套先验做匹配
- **对比**：与九号"退桩前要 RTK = 4"是不同的解决思路 — 九号靠强信号、Luba 靠局部多点定位

### F-T-020 信号差场景退桩持续后退策略（车库场景）

- **场景 / 专题**：信号差场景退桩
- **涉及竞品**：
  - 松灵 Luba：常见用户将 Luba 装在车库内等信号不好的情况，**Luba 会在退桩后一直退**（摄像头朝前、屁股朝后，对它而言是正向走），直到成功 [产品反馈 2026-04-18 §2.2]
- **现象解释**：靠"持续后退 = 视觉持续向前看 = 持续观测桩前环境"来等待信号恢复或视觉建立可信定位
- **关联**：与 F-T-019 是一套连贯的退桩信号差策略
