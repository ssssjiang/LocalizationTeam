## 2. 推荐下一轮测试项（含优先级）

> 优先级判据：(a) 直接关系定位算法决策（融合策略、回环、重定位）权重高；(b) 已有竞品对照数据可直接横评的优先；(c) 已有方案文档但未实施的优先。

### P0 — 必做（高 ROI / 直接关系核心算法）

#### NR-P0-1 RTK 良好 / 阴影区 × 搬运姿态 重定位续建矩阵

- **测试目标**：横向对照九号 i206 在 RTK 良好 / 阴影区下的搬运 1m + 朝向一致 / 偏 90° + 转 3 圈共 5 个场景的续建表现，验证我方融合策略在同等场景下的鲁棒性与精度。
- **推荐场景**：完全复用 [26.3 §2.4] 的 5 个子场景；视频 + 轨迹截图 + 生成图三者齐全。
- **涉及方向**：fusion / vision
- **优先级理由**：竞品已有最严谨的实测数据可直接对标；这是定位算法续建鲁棒性的核心评估。
- **关联缺口**：GAP-01

#### NR-P0-2 RTK 持续信号差下"机身 GPS 跳变兜底"假说验证

- **测试目标**：复现九号 X3 盖锅子场景下"信号差时定位周期性跳变 → 信号好时不主动校准"的现象，验证是否真为机身全频单点 GPS 兜底；同时评估我方融合策略在持续信号差场景下的退化行为。
- **推荐场景**：复用 [26.1 §Navimow X3 - RTK 无信号逻辑探索] 的 4 个子场景（局部建图盖锅子 / 完整建图盖锅子比对老图 / 建图盖锅子+割草不盖锅子 / 割草中盖锅子）。
- **涉及方向**：fusion
- **优先级理由**：直接影响"信号差兜底策略"设计决策；现有 `specials/rtk-startup/` 专题缺这块兜底逻辑实证。
- **关联缺口**：GAP-04、GAP-13

#### NR-P0-3 激光开阔场景回环精度横评（Luba 偏 2 个机身基准）

- **测试目标**：以 Luba 在开阔场景偏 ~2 个机身、机器无法自动闭合为基准，对照我方激光建图在同样开阔场景的闭合误差。
- **推荐场景**：开阔场（无明显特征点 / 无围墙近景），遥控走一个 30-50m 边长的环，回到原点比对；要求**多次重复** ≥3 次取统计。
- **涉及方向**：laser
- **优先级理由**：Luba 已暴露面阵激光在开阔场景的根本性短板；我方需明确自身边界。
- **关联缺口**：GAP-07、GAP-13、GAP-14

#### NR-P0-4 nRTK 网络中断容忍时长基线对齐

- **测试目标**：对照 Yuka 的 ~23min（断 4G & Wi-Fi 后）容忍基线，量化我方 nRTK 路线的中断容忍上限与降级行为。
- **推荐场景**：割草过程中切断 4G + Wi-Fi，记录到首次报错或降级的时间；区分 RTK 阴影区 vs 开阔场。
- **涉及方向**：fusion
- **优先级理由**：竞品基线明确；网络抖动是海外实际部署常见问题。
- **关联缺口**：GAP-03

### P1 — 应做（中等 ROI / 横评类）

#### NR-P1-1 nRTK ↔ RTK 热切换行为对照

- **测试目标**：对照 Yuka "切换不需重建图、有 UI 标识、不动态切" 的特性，验证我方 nRTK 路线的热切换支持。
- **涉及方向**：fusion
- **关联缺口**：GAP-02

#### NR-P1-2 出界距离阈值横评

- **测试目标**：对照九号 ~3m / Yuka Mini ~1.2m 的出界报错阈值与恢复阈值，明确我方边界一致性策略的对外位置。
- **推荐场景**：[26.3 §15] 复用慢推 + 抬后轮拉拽两类操作。
- **涉及方向**：fusion / laser
- **关联缺口**：GAP-05

#### NR-P1-3 寻桩位移容忍横评

- **测试目标**：对照九号 1.5-2m 的寻桩位移容忍 + Luba 红外可对接但地图不更新的逻辑，明确我方"地图位置 vs 实际位置"一致性策略。
- **涉及方向**：fusion / laser
- **关联缺口**：GAP-06

#### NR-P1-4 通道激光建图定位偏差对照

- **测试目标**：对照 Luba 在通道场景下"App 显示位置与实际偏 1-2 个机身、保存通道失败"的现象，验证我方通道专题的定位一致性。
- **推荐场景**：[方案 §6] 窄通道方案直接套用。
- **涉及方向**：laser / fusion
- **关联缺口**：GAP-08、GAP-16

#### NR-P1-5 建图过程中搬运 / 拖拽 / 8 字闭合容忍

- **测试目标**：对照 Luba "支持搬运 1m / 抬起拉拽，但不支持 8 字 / 闭合误检会失败"的边界，验证我方建图引擎的容忍边界。
- **涉及方向**：laser / fusion
- **关联缺口**：GAP-09

#### NR-P1-6 退桩 / 通道建图入口策略对照（**新**）

- **测试目标**：对照九号"退桩建图要 RTK = 4"+"通道建图要重定位成功"+"回桩通道仅 1 条"三条入口策略，与 Luba"退桩三点定位"+"信号差持续后退"两条退桩策略，明确我方策略选型。
- **推荐场景**：
  1. 强信号场景下退桩 / 通道建图（基线）
  2. RTK 信号差（盖锅子等）下尝试退桩 / 通道建图，看我方是否门控
  3. 车库 / 强遮挡场景下退桩，记录退桩动作模式与是否成功
  4. 多回桩通道拓扑下回桩选择
- **涉及方向**：fusion / vision
- **优先级理由**：这三条入口策略直接决定"建图入口阶段是否会建出不可用的图"，是与已确认 W-05（X3 RTK 差时建图不可用）相互验证的关键。
- **关联缺口**：GAP-18、GAP-19、GAP-20、GAP-22

#### NR-P1-7 RTK 差 + 搬运耦合下回桩失效注入测试（**新**）

- **测试目标**：复现九号"RTK 信号特别不好时大部分能正常回充，但发生搬运等情况偶尔会方向混乱出界"的耦合失效模式，量化我方在该耦合下的失效概率与失效模式。
- **推荐场景**：
  1. RTK 阴影区 + 回桩中途搬运 1m + 朝向偏 90°
  2. 多次重复（≥10 次）取统计
- **涉及方向**：fusion
- **优先级理由**：单一信号差不致命，但耦合失效是实际部署中的高频投诉路径；现有 `pause-resume-reloc` 未覆盖该耦合。
- **关联缺口**：GAP-21

## 3. 竞品弱点场景（验证自身优劣的重点）

> 这些是竞品在本批材料里**已明确暴露**的失败/退化场景。建议在我方测试时主动构造这些场景拉对比，作为"我们的优势论据"或"风险预警"。

### W-01 松灵 Luba — 开阔场景面阵激光定位失效

- **失败现象**：建图回到原点偏 ~2 个机身、机器无法自动闭合
- **场景**：开阔场无明显近距特征
- **来源**：[海外分享]、[26.1]
- **建议利用**：我方激光路线在同场景下若能自动闭合，直接构成核心优势论据

### W-02 松灵 Luba — 通道场景定位偏差导致建通道失败

- **失败现象**：建通道时遥控到基座面前 App 仍差 1-2 机身、设置通道起点失败
- **场景**：含基座的通道
- **来源**：[26.1]
- **建议利用**：构造同场景验证我方通道专题表现

### W-03 松灵 Luba — 闭合误检 + 不支持 8 字

- **失败现象**：建图中搬运若触发闭环检测则报"定位丢失"建图失败；不支持 8 字
- **来源**：[26.3 §2.4 Luba nRTK 款]、[方案 §4.1]
- **建议利用**：8 字型场地是重要差异化点

### W-04 九号 i206 — RTK 阴影区下视觉优化无效

- **失败现象**：阴影区搬运后定位不更新、EFLS 视觉优化前后地图基本无变化、整体偏 1m 不可用
- **来源**：[26.3 §2.4]
- **建议利用**：在 RTK 阴影区构造同场景对照我方融合策略

### W-05 九号 X3 — RTK 信号差时建图后地图整体不可用

- **失败现象**：盖锅子建图，信号好后割草整体偏差 1-2m、"实际不能用"；划禁区也掉
- **来源**：[26.1]、[海外分享]
- **建议利用**：验证我方信号差时是否会"建出不可用的图"

### W-06 Sunseeker X7 / X9 — 边界 / 禁区不严格

- **失败现象**：建图离边界坑 ≥5cm 仍掉、画禁区也掉
- **来源**：[26.1 §Sunseeker X7]
- **建议利用**：验证我方边界一致性

### W-07 松灵 Yuka — nRTK 4G 强绑定

- **失败现象**：4G 过期 → nRTK 失效；4G/Wi-Fi 不动态切换
- **来源**：[26.1]、[海外分享]
- **建议利用**：验证我方 nRTK 路线网络降级策略

### W-08 九号 / 松灵 — AI 自主建图边界模糊即失败

- **失败现象**：边界稍模糊 AI 建图就转手动；速度慢、不可靠
- **来源**：[26.1]、[26.3]、[海外分享]
- **建议利用**：模糊边界场景是 AI 建图的共性短板

### W-09 松灵 Luba — 充电桩位移后地图不更新

- **失败现象**：红外能对接到就能回桩，但地图位置不随实际更新
- **来源**：[26.1]
- **建议利用**：地图与物理一致性是验证差异点

### W-10 九号 i108 — 扭矩检测策略不可靠

- **失败现象**：人在屁股后拉机器扭矩检测不出来；推测主要靠单目避障 + 过流 + 打滑
- **来源**：[26.1]、[海外分享]
- **建议利用**：与定位关系较弱，但若我方采用扭矩做打滑兜底，需评估同场景

### W-11 九号 — RTK 差 + 搬运耦合下回桩方向混乱出界（**新**）

- **失败现象**：RTK 信号特别不好时大部分能正常回充，但发生搬运等情况偶尔会方向混乱出界
- **场景**：RTK 阴影区 × 物理扰动叠加
- **来源**：[产品反馈 2026-04-18 §1.3]
- **建议利用**：构造同场景验证我方融合策略在朝向估计退化时的回桩鲁棒性；可作为我方 W-04（i206 阴影区视觉优化无效）的后续耦合实验

### W-12 九号 — 回桩通道仅支持 1 条（拓扑约束，**新**）

- **失败现象**：回桩通道只支持一条充电桩↔草坪的通道；用户多入口需求受限
- **来源**：[产品反馈 2026-04-18 §1.1]
- **建议利用**：若我方支持 N 条回桩通道（如对照 Luba 的 2 条），构成产品差异化论据；同时验证多通道下的回桩选择算法
